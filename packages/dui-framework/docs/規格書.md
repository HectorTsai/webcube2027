# @dui/framework 規格書

> 最後更新：2026-07-20

---

## 1. 套件概述

`@dui/framework` 是 WebCube2027 管理型 Gateway 的**共用框架依賴管理**套件。負責統一提供：

- HTTP 框架（Hono）
- 檔案路由系統（fileRouter）
- CSS 框架（TailwindCSS v4 + daisyUI v5）

管理型 Gateway（ai、page、cube、theme、media）共用此套件，確保依賴版本一致，不需各自管理。

---

## 2. 職責範圍

### 2.1 在範圍內

- 統一匯出 Hono 核心與擴充
- 統一匯出檔案路由系統（fileRouter）
- 管理 TailwindCSS 與 daisyUI 的依賴版本

### 2.2 不在範圍內

- 不包含業務邏輯
- 不包含路由定義（路由由各 Gateway 自行定義）
- 不包含樣式配置（tailwind.config、daisyUI 主題等由各 Gateway 自行配置）

---

## 3. 匯出項目

### 3.1 `mod.ts` — 主要入口

```typescript
export { Hono } from "hono";
export { serveStatic } from "hono/serve-static";
```

| 匯出名稱 | 來源 | 說明 |
|---------|------|------|
| `Hono` | `hono` | Web 框架核心類別 |
| `serveStatic` | `hono/serve-static` | 靜態檔案服務中介軟體 |

### 3.2 `fileRouter.ts` — 檔案路由系統

```typescript
export { createFileRouter } from "@dui/framework/file-router";
```

| 匯出名稱 | 來源 | 說明 |
|---------|------|------|
| `createFileRouter` | 自製 | 掃描 routes/ 目錄，自動註冊 handler 與靜態檔案 |

**支援的檔案類型：**

| 類型 | 副檔名 | 行為 |
|------|--------|------|
| Method handler | `get.ts`, `post.ts`, `put.ts`, `del.ts`, `patch.ts` | 匯出對應 method 的 handler 函數 |
| 靜態檔案 | `.css`, `.svg`, `.jpg`, `.png`, `.webp`, `.js` | 自動以 GET 服務，含正確 Content-Type |
| 中間件 | `_middleware.ts` | 匯出 `middleware`（或 `default`）函數，套用至該目錄所有路由 |

**目錄命名規則：**
- `foo/` → 靜態路徑段落 `/foo`
- `_name_/` → 動態參數 `/:name`

## 4. 管理依賴

| 依賴 | 版本 | 用途 | 消費方式 |
|------|------|------|---------|
| `hono` | ^4.12.31 | Web 框架 | workspace 根目錄 `imports` 全域共用 |

| `@tailwindcss/cli` | ^4.3.3 | TailwindCSS CLI | 同上 |
| `daisyui` | ^5.6.18 | UI 元件庫 | 此套件管理版本 |

## 5. 使用方式

### HTTP 服務

```typescript
import { Hono } from "@dui/framework";

const app = new Hono();
```

### 檔案路由

```typescript
import { createFileRouter } from "@dui/framework/file-router";

const fileRoutes = await createFileRouter({ dirPath: "./routes" });
app.route("/", fileRoutes);
```

`routes/` 目錄範例：

```
routes/
├── admin/
│   ├── _middleware.ts   ← 目錄守衛
│   └── get.ts           → GET /admin
├── api/
│   └── _model_/         → /api/:model
│       ├── get.ts       → GET /api/:model
│       └── _id_/
│           └── get.ts   → GET /api/:model/:id
├── logo.svg             → GET /logo.svg
└── styles/
    └── theme.css        → GET /styles/theme.css
```

## 7. 目錄結構

```

├── mod.ts                  ← 主要入口（Hono + serveStatic）
├── fileRouter.ts           ← 檔案路由系統

├── deno.json               ← 套件設定 + 依賴管理
└── docs/
    └── 規格書.md           ← 本文件
```
