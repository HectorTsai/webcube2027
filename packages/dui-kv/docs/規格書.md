# @dui/kv 規格書

> 最後更新：2026-07-22

---

## 1. 套件概述

`@dui/kv` 是 WebCube2027 的**輕量級持久化 Key-Value Store**（L1 設定儲存層）。以 JSON 檔案為後端，提供簡單可靠的 KV 讀寫介面。

**定位**：

- 僅存放系統設定、L2 資料庫連線資訊等設定資料
- **不是完整資料庫**，不參與查詢路由或資料降級
- 無 heavy dependencies（不依賴 mongodb、mysql、pg 等資料庫驅動）

**設計目標**：

- 極簡依賴：只依賴 `@dui/util`（logger）
- 零配置啟動：首次使用自動建立資料目錄與檔案
- 即時持久化：每次 `set()` / `delete()` 立即 flush 到磁碟
- 人性化儲存：JSON 格式，可直接開啟編輯或除錯

---

## 2. 相依關係

```
@dui/kv
  └── @dui/util    ← 僅日誌（info / debug / error）
```

與 `@dui/database` 的關係：
- `@dui/database` 在啟動時透過 `dataPool.setConfigStore(l1)` 取得 L1Store 參考
- `@dui/kv` 本身不知道 Pool 的存在，保持純粹的 KV 儲存職責

## 3. L1Store 類別

### 3.1 建構子

```typescript
constructor(dataPath?: string)
```

| 參數 | 型別 | 預設值 | 說明 |
|------|------|--------|------|
| `dataPath` | `string` (optional) | `Deno.env.get('L1_PATH') ?? './data/l1.json'` | JSON 檔案儲存路徑 |

### 3.2 核心方法

| 方法 | 簽名 | 說明 |
|------|------|------|
| `init()` | `(): Promise<void>` | 初始化 — 確保資料目錄存在，載入既有資料到記憶體快取 |
| `get(key)` | `(key: string): Promise<string \| null>` | 取得值，不存在回傳 `null` |
| `set(key, value)` | `(key: string, value: string): Promise<void>` | 設定值，立即 flush 到磁碟 |
| `delete(key)` | `(key: string): Promise<void>` | 刪除鍵，立即 flush 到磁碟 |
| `has(key)` | `(key: string): boolean` | 檢查鍵是否存在（純記憶體操作） |
| `keys()` | `(): string[]` | 取得所有鍵 |
| `exportAll()` | `(): Record<string, string>` | 匯出所有資料為純物件（用於備份/管理介面顯示） |

### 3.3 內部機制

**資料載入（`load()`）**：
- 讀取 JSON 檔案，僅載入 `string` 型別的值
- 非字串值、非物件、陣列等格式錯誤皆忽略
- 檔案不存在或解析失敗時，以空快取啟動

**資料寫入（`flush()`）**：
- 每次 `set()` / `delete()` 立即將記憶體快取序列化為 JSON 寫入磁碟
- 使用 `JSON.stringify(obj, null, 2)` 產生人類可讀格式

**目錄建立（`ensureDir()`）**：
- 自動建立 JSON 檔案所在的目錄（`mkdir recursive`）
- 首次使用不需手動建立資料目錄

### 3.4 使用範例

```typescript
import { L1Store } from '@dui/kv';

// 使用預設路徑 ./data/l1.json
const store = new L1Store();
await store.init();

// 讀寫
await store.set('theme', 'dark');
const theme = await store.get('theme');  // "dark"

// 檢查與刪除
if (store.has('theme')) {
  await store.delete('theme');
}

// 匯出所有
const all = store.exportAll();  // { "key1": "value1", ... }

// 指定路徑
const custom = new L1Store('./config/settings.json');
await custom.init();
```

## 4. 資料格式

`l1.json` 檔案格式範例：

```json
{
  "l2_connection": "U2FsdGVkX1...",
  "theme": "dark",
  "language": "zh-TW",
  "_crypto_key": "a1b2c3d4e5f6..."
}
```

- 所有值為**字串型別**
- 敏感資料（如 L2 連線資訊）應預先加密再存入
- `_crypto_key` 保留給 `@dui/framework` 管理加密金鑰

---

## 5. 與 `@dui/framework` 的整合

`@dui/kv` 通常不直接被 Gateway 使用，而是透過 `@dui/framework` 的 `createGateway()` 間接取得：

```typescript
// Gateway main.ts — 不直接 import @dui/kv
import { createGateway } from '@dui/framework';

const gw = await createGateway({
  name: 'my-gateway',
  port: 8000,
  dirname: import.meta.dirname!,
});

// gw.l1 即為已初始化的 L1Store 實例
await gw.l1.get('my_key');
await gw.l1.set('my_key', 'my_value');
```

若需要直接使用（如在其他非 Gateway 場景），亦可直接 import：

```typescript
import { L1Store } from '@dui/kv';
```
