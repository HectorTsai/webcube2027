# @dui/util 規格書

> 最後更新：2026-07-20

---

## 1. 套件概述

`@dui/util` 是 WebCube2027 的通用工具套件，提供跨所有 Gateway 共用的基礎設施：

- **InnerAPI** — 內部服務間通訊代理（附 Request 級別快取）
- **Logger** — 日誌記錄工具（含層級過濾與檔案輸出）
- **Crypto** — AES-256-GCM 加解密工具
- **Response** — 統一 API 回應型別與建構函式

---

## 2. InnerAPI — 內部 API 代理

### 2.1 概述

InnerAPI 是 Gateway 之間進行 HTTP 內部通訊的代理機制。所有管理型 Gateway 透過 InnerAPI 向 `data-gateway` 發起資料請求，不直接暴露外部埠號。

核心特性：**Request 級別透明快取**，同一個請求在相同 Context 內只會執行一次。

### 2.2 方法

| 方法 | 說明 |
|------|------|
| `設定App(app)` | 啟動時注入 Hono app 實例，供 InnerAPI 內部路由使用 |
| `InnerAPI(c, apiPath)` | 發起內部 API 請求，返回 `Promise<Response>` |
| `取得域名(c)` | 從 Context 解析目前請求的 hostname（不含埠號） |

### 2.3 快取機制

```
InnerAPI(c, "/api/settings")
  → 檢查 c.get("api_internal_cache") 是否有對應路徑
    → 命中：直接回傳快取 Response（header X-Cache: HIT-REQUEST-CONTEXT）
    → 未命中：執行內部請求 → 快取結果 → 回傳 Response

快取生命週期：僅限單一 HTTP Request 的 Context
同一請求內多次呼叫相同路徑只會查詢一次資料庫
```

### 2.4 使用範例

```typescript
import { Hono } from "hono";
import { InnerAPI, 設定App } from "@dui/util";

const app = new Hono();
設定App(app);  // 啟動時注入

app.get("/page", async (c) => {
  const data = await InnerAPI(c, "/api/網站資訊/example.com");
  // 在 render-gateway 內，這會呼叫 data-gateway 的內部路由
  return c.json(data);
});
```

### 2.5 注意事項

- 必須在 `main.ts` 啟動流程中呼叫 `設定App(app)`，否則 InnerAPI 會拋出錯誤
- 內部請求會自動帶入原始請求的 `cookie`、`host`、`origin` header
- 非 JSON 回應不會被快取

## 3. Logger — 日誌記錄

### 3.1 概述

Logger 提供統一的日誌記錄介面，支援層級過濾與選擇性檔案寫入。

### 3.2 環境變數配置

| 變數 | 預設值 | 說明 |
|------|--------|------|
| `LOG_LEVEL` | `info` | 日誌過濾層級：`debug` / `info` / `warn` / `error` |
| `LOG_TO_FILE` | `false` | 設為 `true` 啟用檔案輸出，寫入 `./logs/app.log` |

### 3.3 方法

| 方法 | 說明 |
|------|------|
| `logger.debug(模組, 訊息)` | 除錯日誌 |
| `logger.info(模組, 訊息)` | 一般資訊 |
| `logger.warn(模組, 訊息)` | 警告 |
| `logger.error(模組, 訊息)` | 錯誤 |
| `debug(模組, 訊息)` | 全域便捷函式（呼叫 `logger.debug`） |
| `info(模組, 訊息)` | 同上，對應 `logger.info` |
| `warn(模組, 訊息)` | 同上，對應 `logger.warn` |
| `error(模組, 訊息)` | 同上，對應 `logger.error` |

### 3.4 輸出格式

```
[2026-07-20 14:30:00] [INFO] [模組名稱] 訊息內容
```

層級過濾規則：低於設定層級的日誌不會輸出（例如 `LOG_LEVEL=info` 時，`debug` 層級不會顯示）。

### 3.5 使用範例

```typescript
import { info, error, logger } from "@dui/util";

info("MyGateway", "服務啟動完成");
error("MyGateway", "資料庫連線失敗");

// 直接使用 logger 實例
logger.warn("MyGateway", "記憶體使用率偏高");
```

## 4. Crypto — 加解密工具

### 4.1 概述

提供 AES-256-GCM + PBKDF2 加解密功能，用於保護 L1 連線資訊等敏感資料。

金鑰解析順序：
1. **`SECRET_KEY` 環境變數**（production 佈署用）
2. **`./data/.crypto.key` 檔案**（第一次執行自動產生，重啟後依然有效）
3. **自動產生** — 256-bit 隨機金鑰，寫入 `./data/.crypto.key`（權限 600）

### 4.2 方法

| 方法 | 說明 |
|------|------|
| `encrypt(value)` | 加密字串，回傳 `enc:v1\|...` 格式的 Base64 字串 |
| `decrypt(value)` | 解密字串，非 `enc:v1` 前綴的值視為明文穿透 |
| `ensureKey()` | 確保加密金鑰存在（啟動時呼叫，不觸發加解密） |

### 4.3 使用範例

```typescript
import { encrypt, decrypt, ensureKey } from "@dui/util";

// 啟動時確保金鑰
ensureKey();

// 加密
const encrypted = await encrypt("敏感資料");

// 解密
const decrypted = await decrypt(encrypted);
```

### 4.4 SecretString

`SecretString` 是一個自動加解密的字串包裝器，適合 Model 中儲存密碼或金鑰：

```typescript
import { SecretString } from "@dui/util";

const secret = new SecretString({ plainText: "my-api-key" });
const plain = await secret.getPlainText();   // "my-api-key"
const enc = await secret.getCipherText();    // "enc:v1|..."
```

## 5. Response — 統一 API 回應

### 5.1 概述

提供所有 Gateway 共用的 API 回應型別與建構函式。不含 Hono 依賴，純資料層級。

### 5.2 型別

| 型別 | 說明 |
|------|------|
| `ErrorCode` | 錯誤代碼常數（`NOT_FOUND`, `FORBIDDEN`, `VALIDATION_ERROR`, `BAD_REQUEST`, `INTERNAL_ERROR`） |
| `SourceType` | 資料來源層級（`'L2'` \| `'L3'`） |
| `SuccessResponse<T>` | 成功回應：`{ success: true, data, source }` |
| `ErrorResponse` | 錯誤回應：`{ success: false, error: { code, message, status } }` |
| `ApiResponse<T>` | 聯集型別：`SuccessResponse<T> \| ErrorResponse` |
| `PaginationInfo` | 分頁資訊：`{ page, pageSize, total, totalPages }` |
| `PaginatedData<T>` | 分頁資料：`{ items, pagination }` |

### 5.3 建構函式

| 函式 | 說明 |
|------|------|
| `success(data, source?)` | 建立成功回應 |
| `paginated(items, pagination, source?)` | 建立分頁回應 |
| `errorRes(code, message, status)` | 建立錯誤回應 |
| `Errors.notFound(msg?)` | 404 錯誤便利建構 |
| `Errors.forbidden(msg?)` | 403 錯誤便利建構 |
| `Errors.validationError(msg?)` | 422 錯誤便利建構 |
| `Errors.badRequest(msg?)` | 400 錯誤便利建構 |
| `Errors.internalError(msg?)` | 500 錯誤便利建構 |

### 5.4 使用範例

```typescript
import { success, paginated, Errors } from "@dui/util";

// 成功回應
return c.json(success({ id: "1", name: "foo" }));

// 分頁回應
return c.json(paginated(items, { page: 1, pageSize: 20, total: 100, totalPages: 5 }));

// 錯誤回應
return c.json(Errors.notFound("使用者不存在"), 404);
```
