## ❌ 核心禁令 (Anti-Patterns)
1. **嚴格禁止 Hard-coding（硬編碼）**：所有的 API 網址、Port、第三方 Token 與資料庫連線字串，必須使用 Deno 原生的 `Deno.env.get("KEY")` 或 Hono 的 `env(c)` 讀取，絕對不能直接寫死在程式碼中。
2. **禁止 Node.js 污染**：本專案是 Deno 專案，禁止使用 `process.env`、`require()` 或引入 `npm:` 以外的 Node 原生套件。統一使用 Deno 標準庫 or 專屬 Deno 第三方模組。
3. **禁止大型 JS 框架思維**：不使用 React/Vue 的虛擬 DOM 思維。所有前端邏輯完全交給 Alpine.js 與 Hono 的 JSX/Html 模板處理。
4. **禁止直接使用 Console**：全專案嚴禁使用 `console.log`，必須統一使用 `utils/logger.ts`。

## 🗂️ 多租戶與動態 JSON 渲染架構規範
1. **多租戶與多資料庫安全**：本專案為多租戶系統，支援租戶自主選擇資料庫類型（包含 3 層資料庫架構與動態資料庫路由）。所有資料庫操作與連線配置必須使用抽象層（Adapter/Repository 模式）處理，嚴格禁止寫死任何特定資料庫語法。
2. **JSON 動態渲染規則**：所有 AI 生成的網頁、佈景主題與 UI 元件，必須完全符合專案定義的標準 JSON Schema 格式並儲存於資料庫中。在編寫相關的渲染器（Renderer）與 API 時，必須嚴格遵循動態加載原則。
