# UI 元件系統設計

## 概述

WebCube2027 的 UI 元件系統採用模組化設計，支援多種風格和自定義選項，提供一致且靈活的使用者介面。

## 設計原則

### 1. 模組化架構
- **風格分離**：每種風格獨立實作
- **統一介面**：所有風格使用相同的 props 介面
- **易於擴展**：新增風格只需要建立新目錄

### 2. 後備機制
- **優先載入**：嘗試載入指定風格
- **優雅降級**：載入失敗時使用預設風格
- **穩定性**：確保總有可用的元件

### 3. TypeScript 支援
- **型別安全**：完整的 props 型別定義
- **自動完成**：編輯器提示支援
- **錯誤檢查**：編譯時型別檢查

## 按鈕元件

### 設計決策

```typescript
// src/components/按鈕.tsx
interface ButtonProps {
  children: string
  顏色?: '主要' | '次要' | '成功' | '警告' | '錯誤' | '資訊'
  尺寸?: '小' | '中' | '大'
  禁用?: boolean
  風格: string // 必要參數，從 route 傳入
  type?: 'button' | 'submit' | 'reset'
  onClick?: () => void
  className?: string
}

export default async function Button({ 風格, ...props }: ButtonProps) {
  try {
    // 1. 嘗試從檔案系統載入
    const Button = (await import(`./${風格}/Button.tsx`)).default
    return <Button {...props} />
  } catch (_e) {
    // 2. 後備方案：使用預設實心按鈕
    return <實心按鈕 {...props} />
  }
}
```

### 為什麼不支援資料庫？

- **按鈕是純 UI 元件**：邏輯簡單，不需要動態內容
- **檔案系統足夠**：預設風格已滿足大部分需求
- **保持簡單**：避免過度設計

### 風格實作範例

#### 實心風格
```typescript
// src/components/實心/Button.tsx
export default function Button({ 
  children, 
  顏色 = '主要', 
  尺寸 = '中', 
  禁用 = false,
  type = 'button',
  onClick,
  className = ''
}: ButtonProps) {
  const 顏色類別 = {
    主要: '背景-主色 文字-主色內容 hover:opacity-90',
    次要: '背景-次色 文字-次色內容 hover:opacity-90',
    成功: '背景-成功色 文字-成功色內容 hover:opacity-90',
    警告: '背景-警告色 文字-警告色內容 hover:opacity-90',
    錯誤: '背景-錯誤色 文字-錯誤色內容 hover:opacity-90',
    資訊: '背景-資訊色 文字-資訊色內容 hover:opacity-90'
  }

  const 尺寸類別 = {
    小: 'px-3 py-1.5 text-sm',
    中: 'px-4 py-2 text-sm',
    大: 'px-6 py-3 text-base'
  }

  const 禁用類別 = 禁用 ? 'opacity-50 cursor-not-allowed' : ''

  return (
    <button
      type={type}
      disabled={禁用}
      onClick={onClick}
      className={`
        inline-flex items-center justify-center font-medium 圓角-欄位
        focus:outline-none transition-all duration-200
        ${顏色類別[顏色]}
        ${尺寸類別[尺寸]}
        ${禁用類別}
        ${className}
      `}
    >
      {children}
    </button>
  )
}
```

## 技術決策記錄

### 動態載入 vs 靜態載入

- **最終決定**：使用動態載入 `import(\`${風格}/Button.tsx\`)`
- **原因**：支援執行時載入不同風格
- **優點**：模組化，按需載入
- **缺點**：需要錯誤處理

### 資料庫依賴

- **按鈕元件**：不依賴資料庫，保持簡單
- **載入器元件**：預留資料庫介面，但不強制使用

#### 多層安全檢查
```typescript
// 1. 語法驗證
function syntaxValidation(component) {
  const dangerousPatterns = [
    /eval\s*\(/,
    /Function\s*\(/,
    /document\./,
    /window\./,
    /localStorage/,
    /fetch\s*\(/,
    /javascript:/,
    /on\w+=/
  ]
  
  const componentString = JSON.stringify(component)
  const hasDanger = dangerousPatterns.some(pattern => pattern.test(componentString))
  
  return { safe: !hasDanger, issues: hasDanger ? ['偵測到危險語法'] : [] }
}

// 2. AI 語意檢查
async function aiSecurityCheck(component) {
  const prompt = `
  請檢查以下元件配置的安全性：
  ${JSON.stringify(component, null, 2)}
  
  回傳 JSON 格式：
  {
    "safe": true/false,
    "risks": ["風險1", "風險2"],
    "suggestions": ["建議1", "建議2"]
  }
  `
  
  return await aiSecurityAnalysis(prompt)
}

// 3. 沙盒測試
async function sandboxTest(component) {
  // 在隔離環境中測試元件渲染
  const sandbox = createSecureSandbox()
  try {
    const result = await sandbox.render(component)
    return { safe: true, result }
  } catch (error) {
    return { safe: false, error: error.message }
  }
}
```

### 👥 審核機制

#### 審核流程
```typescript
class ComponentReviewSystem {
  async submitForReview(submission: ComponentSubmission): Promise<void> {
    // 1. 自動化檢查
    const autoChecks = await this.runAutomatedChecks(submission)
    
    // 2. 分配審核者
    const reviewer = await this.assignReviewer(submission)
    
    // 3. 通知審核者
    await this.notifyReviewer(reviewer, submission)
    
    // 4. 更新狀態
    submission.status = 'under_review'
    submission.assignedReviewer = reviewer.id
    await this.updateSubmission(submission)
  }
  
  async approveComponent(submissionId: string, reviewNotes: string): Promise<void> {
    const submission = await this.getSubmission(submissionId)
    
    // 1. 標記為已發布
    submission.status = 'approved'
    submission.approvedAt = new Date()
    submission.reviewNotes = reviewNotes
    
    // 2. 更新版本號
    submission.version = this.generateNewVersion()
    
    // 3. 加入元件庫
    await this.addToComponentLibrary(submission)
    
    // 4. 通知相關使用者
    await this.notifyApproval(submission)
    
    // 5. 記錄審核日誌
    await this.logReviewAction(submission, 'approved')
  }
  
  async rejectComponent(submissionId: string, reason: string): Promise<void> {
    const submission = await this.getSubmission(submissionId)
    
    submission.status = 'rejected'
    submission.rejectionReason = reason
    submission.rejectedAt = new Date()
    
    await this.updateSubmission(submission)
    await this.notifyRejection(submission)
    await this.logReviewAction(submission, 'rejected')
  }
}
```
