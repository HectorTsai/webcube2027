# 方塊設計指南（AI 閱讀用）

> 版本：2026-06 / 最後更新：移除方塊定義 + text 併入 children + 支援直寫 prop

---

## 1. 核心概念

**方塊** 是 webcube2027 的資料驅動元件系統。每個方塊是一筆 JSON，描述一個 UI 節點的結構、樣式、行為。`Cube.tsx` 是萬用渲染器，讀取方塊定義後渲染成對應的 HTML。

有兩種使用方式，本質等價：

| 方式 | 場景 | 範例 |
|---|---|---|
| **JSX children** | 靜態頁面、測試檔 | `<Cube definition={d}>...</Cube>` |
| **資料驅動** | 資料庫 seed、API 回傳 | `{ from: "div", children: [...] }` |

---

## 2. 方塊定義 JSON 欄位

```typescript
// database/models/方塊.ts
type 方塊欄位 = {
  // ── 必填 ──
  from: string;          // "div" | "span" | "方塊:方塊:Container" | ...
  className: string;     // 預設 CSS class（預設 ""）
  style: Record<string, string>;  // 預設 inline style（預設 {}）
  args: Record<string, ArgDef>;   // 參數定義（預設 {}）
  售價: number;          // 預設 0

  // ── 可選 ──
  alpine?: Record<string, unknown>;  // { init, bind, model }
  on?: Record<string, string>;       // { click: "...", input: "..." }
  data?: Record<string, string>;     // { key: "value" } → data-key="value"
  slots?: Record<string, unknown>;   // 具名插槽 { head: {...}, body: {...} }
  children?: (Record<string, unknown> | string)[];  // 子方塊或純文字
};
```

### ArgDef 參數定義

```typescript
interface ArgDef {
  type: "string" | "number" | "boolean";
  description?: string;
  default?: unknown;
  options?: string[];  // 字串類型可指定列舉值
}
```

### from 的兩種值

| from 值 | 行為 |
|---|---|
| `"div"` `"span"` `"input"` ... | 原生 HTML 標籤，Cube 直接渲染 |
| `"方塊:方塊:Container"` 等 | 方塊 ID，查 fallback registry，委派給對應元件 |

### alpine 子欄位

- `init`: Alpine.js `x-init` 字串
- `bind`: `{ key: "expression" }` → 多個 `x-bind:key`
- `model`: `x-model` 字串

## 3. Cube.tsx 使用方式

### 靜態 JSX（推薦寫法）

```tsx
import Cube from '../components/方塊.tsx';

// 基本原生標籤
<Cube definition={{ from: "div", className: "p-4" }}>
  hello
</Cube>

// Container fallback + 直寫 prop
<Cube definition={containerDef} color="primary" active={true} hover={false}>
  <Cube definition={{ from: "span", className: "font-bold" }}>
    內容
  </Cube>
</Cube>
```

### 資料驅動（seed / API）

```json
{
  "from": "div",
  "className": "p-4 space-y-2",
  "children": [
    "純文字節點",
    { "from": "span", "className": "font-bold", "children": ["粗體文字"] }
  ]
}
```

### 參數合併優先序

```
defaults（ArgDef.default） → args prop（物件） → 直寫 prop
```

三者會合併，後者覆蓋前者。例如：

```tsx
// defaults: { color: "primary", active: true }
// 最終 mergedArgs: { color: "error", active: false }
<Cube definition={d} color="error" active={false} />
```

## 4. Fallback 機制（內建元件）

當 `from` 不是原生標籤時，Cube 會查 `fallbackRegistry`：

```typescript
// components/Cube.tsx
registerFallback("方塊:方塊:Container", (props) => (
  <Container {...props} />
));
```

註冊後，任何 Cube 使用 `from: "方塊:方塊:Container"` 都會委派給 `Container.tsx`。也可以透過 `CubeProps.fallbacks` 臨時覆蓋。

---

## 5. Seed 資料格式

Seed 放在 `database/seeds/方塊/` 下，可為單一 JSON 檔或目錄（遞迴掃描）。

```json
// database/seeds/方塊/container.json
{
  "id": "方塊:方塊:Container",
  "名稱": {"en": "Container", "zh-tw": "容器"},
  "描述": {"en": "...", "zh-tw": "..."},
  "from": "方塊:方塊:Container",
  "args": {
    "color": {
      "type": "string",
      "description": "色彩主題",
      "default": "primary",
      "options": ["primary", "secondary", "accent"]
    }
  }
}
```

伺服器啟動時若資料庫為空，`讀取種子("方塊")` 會掃描該目錄，每筆 JSON 傳入 `new 方塊(data)` 後寫入 DB。

---

## 6. children 規則

- JSX children 優先於 `definition.children`
- `children` 陣列元素可以是 **物件**（子方塊定義）或 **字串**（純文字）
- 遞迴深度上限 10 層
- `slots` 與 `children` 可並存，都會被渲染
- 最終 children 會經過 `processChildren()` 注入 context

---

## 7. 不要做的事

- **不要** 建立 `方塊定義` interface — 已刪除，統一用 `Partial<方塊>`
- **不要** 使用 `text` 欄位 — 已移除，請用 `children: ["文字"]`
- **不要** 在跟 `from` 無關的元件上寫 `padding` arg — Container 已不處理 padding，用 `className="p-sm"` 取代
- **不要** 把 `_args` 當作有效參數傳給 helper — 移除所有 `makeContainer()` 這類中間層，直接寫 JSX
