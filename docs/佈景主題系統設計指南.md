# 佈景主題系統設計指南（AI 閱讀用）

> 版本：2026-06 / 架構從「骨架 + 配色」擴充為「骨架 + 配色 + 風格 + 動畫 + 裝飾」

---

## 1. 核心架構

**佈景主題** 是一個組合體，由五個獨立模組拼裝而成：

```
佈景主題 = 骨架 + 配色 + 風格 + 動畫 + 裝飾
```

| 模組 | 職責 | ID 前綴 |
|---|---|---|
| **骨架** | 幾何結構、字型尺寸、間距、圓角、佈局 | `骨架:骨架:` |
| **配色** | OKLCH 色彩定義（主色/次色/強調/中性/背景/功能色） | `配色:配色:` |
| **風格** | CSS class 組合拳（active/inactive/hover 三態 + 漸層 + 陰影） | `風格:風格:` |
| **動畫** | UI 互動動畫 class 映射（抽屜/視窗/下拉/彈出） | `動畫:動畫:` |
| **裝飾** | 頁面掛件與 SVG 裝飾（角落徽章、浮水印） | `裝飾:裝飾:` |

每個模組獨立存在於資料庫，佈景主題只存**指針（ID）**，不內嵌內容。這樣一個配色可以被多個佈景主題共用，一個動畫定義也可以跨主題使用。

## 2. 配色

### Model (`database/models/配色.ts`)

色彩值使用 **OKLCH 格式**（`"L C H"` 三個數字，空格分隔）。這是 CSS Color Level 4 標準，支援 P3 廣色域。

```typescript
class 配色 {
  名稱: MultilingualString;
  描述: MultilingualString;
  主色: string;    // --p（primary），例如 "59.67% 0.221 258.03"
  次色: string;    // --s（secondary）
  強調色: string;  // --a（accent）
  中性色: string;  // --nc（neutral-content 基色）
  背景色: string;  // --b1（base-100）
  資訊色: string;  // --info
  成功色: string;  // --success
  警告色: string;  // --warning
  錯誤色: string;  // --error
  售價: number;
}
```

### 如何新增配色

在 `database/seeds/配色.json` 中加入一筆，格式範例：

```json
{
  "id": "配色:配色:深海藍",
  "名稱": {"en": "Deep Ocean", "zh-tw": "深海藍"},
  "描述": {"zh-tw": "深邃的海洋藍調，沉穩內斂。"},
  "主色": "45% 0.25 255",
  "次色": "35% 0.12 250",
  "強調色": "70% 0.15 200",
  "中性色": "30% 0.03 260",
  "背景色": "98% 0.01 260",
  "資訊色": "65% 0.18 240",
  "成功色": "55% 0.14 160",
  "警告色": "70% 0.19 50",
  "錯誤色": "55% 0.23 25",
  "售價": 0,
  "可刪除": false
}
```

## 3. 骨架

### Model (`database/models/骨架.ts`)

骨架掌管三大類資訊：**幾何 Token**（CSS 變數值）、**佈局選擇**（用哪個方塊當佈局）、**元件預設**（圖示集、書本樣式、選單按鈕等）。

```typescript
class 骨架 {
  名稱: MultilingualString;
  描述: MultilingualString;
  影像: string;          // 預覽圖
  佈局: string;           // 佈局方塊 ID，例如 "方塊:方塊:basic-layout"
  圖示: string;           // 圖示集名稱，例如 "外框"
  書本樣式: string;       // 書本翻頁樣式
  選單按鈕: string;       // 選單按鈕圖示 ID
  開始動畫: string;       // loading 動畫名稱
  載入器: string;         // spinner 樣式
  配置: Record<string, string>;  // 幾何 Token（radius/spacing/font/border/icon/image）
  售價: number;
}
```

### 配置 Token 命名規則

所有 Token 由前端程式直接透過 CSS 變數消費。命名規則如下：

| 前綴 | 用途 | 範例 |
|---|---|---|
| `radius-` | 圓角 | `radius-sm` → `0.25rem` |
| `spacing-` | 間距 | `spacing-md` → `1rem` |
| `font-` | 字級 | `font-3xl` → `1.875rem` |
| `leading-` | 行高 | `leading-base` → `1.5rem` |
| `border-` | 邊框 | `border-md` → `1.5px` |
| `icon-` | 圖示尺寸 | `icon-md` → `24px` |
| `image-` | 圖片尺寸 | `image-lg` → `192px` |

擴充新 Token 時，直接在 `配置` 中加 key，不需要改程式碼。

### 如何新增骨架

```json
{
  "id": "骨架:骨架:寬鬆舒適",
  "名稱": {"zh-tw": "寬鬆舒適骨架"},
  "描述": {"zh-tw": "加大間距與圓角，營造呼吸感。"},
  "佈局": "方塊:方塊:basic-layout",
  "配置": {
    "radius-sm": "0.5rem",
    "radius-md": "1rem",
    "radius-lg": "1.5rem",
    "spacing-xs": "0.75rem",
    "spacing-sm": "1rem",
    "spacing-md": "1.5rem",
    "spacing-lg": "2rem",
    "spacing-xl": "3rem",
    "font-base": "1.125rem"
  },
  "售價": 0
}
```

## 4. 風格

### Model (`database/models/風格.ts`)

風格是 **CSS class 組合拳** 的容器。每個 key 對應一組 UnoCSS / Tailwind class 字串，由元件在渲染時動態載入。

```typescript
class 風格 {
  名稱: MultilingualString;
  描述: MultilingualString;
  配置: Record<string, string>;  // key → CSS class 字串
  售價: number;
}
```

### 風格的核心 Key

目前 Container 等元件使用的三態 key：

| Key | 用途 | 觸發時機 |
|---|---|---|
| `active` | 供電狀態的 class | `active={true}` |
| `inactive` | 斷電狀態的 class | `active={false}` |
| `hover` | 懸停效果的 class | `hover={true}` + `:hover` |

其他 key 完全自由擴充。例如 `外框線.json` 的 `配置`：
```json
{
  "active": "bg-transparent text-current border-2 border-solid border-current shadow-sm",
  "inactive": "bg-transparent text-current-30 border-2 border-solid border-current-50/30 shadow-none saturate-30 opacity-50",
  "hover": "hover:bg-current-10/20 hover:text-current-content hover:border-current-70 hover:shadow-md"
}
```

### 與 Container.tsx 的配合

`Container.tsx` 使用 `c-style-apply` + `c-div-active` + `c-div-hover` + `c-div-inactive` 這四個 UnoCSS rule/shortcut，把風格的 active/inactive/hover class 映射到 `[data-active]` 和 `[data-hover]` 屬性選擇器上。風格定義裡的 `bg-current`、`text-current` 等 current token 會被替換成對應的 OKLCH CSS 變數。

### 如何新增風格

```json
{
  "id": "風格:風格:毛玻璃",
  "名稱": {"zh-tw": "毛玻璃"},
  "描述": {"zh-tw": "半透明模糊背景，現代科技感。"},
  "配置": {
    "active": "bg-current-10/40 backdrop-blur-xl border border-white/20 shadow-lg text-current-content",
    "inactive": "bg-base-50/20 backdrop-blur-sm border border-base-50/10 shadow-none opacity-60",
    "hover": "hover:bg-current-20/50 hover:backdrop-blur-2xl hover:shadow-xl"
  },
  "售價": 0
}
```

## 5. 動畫

### Model (`database/models/動畫.ts`)

動畫模組是 **萬能動畫 class 映射水庫**。Key 格式為 `"元件:動作"`，Value 是 UnoCSS 動畫 class 字串。

```typescript
class 動畫 {
  名稱: MultilingualString;
  描述: MultilingualString;
  配置: Record<string, string>;  // "元件:動作" → "animate-in fade-in ..."
  售價: number;
}
```

### 目前的 Key 設計

| Key 格式 | 範例 | 用途 |
|---|---|---|
| `下拉選單:開/關` | `animate-in fade-in` | 下拉選單動畫 |
| `抽屜:上/下/左/右:開/關` | `animate-in slide-in-from-top` | 四向抽屜動畫 |
| `視窗:開/關` | `animate-in zoom-in-95 fade-in` | Modal 視窗動畫 |
| `彈出:開/關` | `animate-in scale-in` | Popup/Tooltip 動畫 |
| `折疊面板:開/關` | `animate-in slideInDown` | 未來擴充範例 |
| `吐司訊息:進場/出場` | `animate-in backInRight` | 未來擴充範例 |

擴充新元件時，直接在 `配置` 中加 key，前端元件用 `動畫.配置["新元件:開"]` 讀取即可，不需要改任何程式碼。

### 如何新增動畫

```json
{
  "id": "動畫:動畫:彈跳風",
  "名稱": {"zh-tw": "彈跳風"},
  "描述": {"zh-tw": "活潑的彈跳動畫，適合活潑品牌。"},
  "配置": {
    "下拉選單:開": "animate-in zoom-in-95 fade-in duration-300",
    "視窗:開": "animate-in bounce-in duration-400",
    "彈出:開": "animate-in bounce-in-up duration-200",
    "彈出:關": "animate-out bounce-out-down duration-150"
  },
  "售價": 10
}
```

## 6. 裝飾

### Model (`database/models/裝飾.ts`)

裝飾模組是 **頁面掛件與 SVG 水庫**。Key 為掛件位置名稱，Value 為圖示 ID（指向 `圖示:圖示:...`）或 `"none"`（不顯示）。

```typescript
class 裝飾 {
  名稱: MultilingualString;
  描述: MultilingualString;
  配置: Record<string, string>;  // 掛件位置 → 圖示 ID | "none"
  售價: number;
}
```

### Key 命名慣例

以 `位置-語意` 命名，例如：
- `container-top-left` / `container-top-right` / `container-bottom-left` / `container-bottom-right`
- `page-watermark` — 全頁浮水印
- `badge-top-left` — 左上角徽章
- `card-ornament` — 卡片裝飾
- `divider-center-icon` — 分隔線中央圖示

### 如何新增裝飾

```json
{
  "id": "裝飾:裝飾:新春喜慶",
  "名稱": {"zh-tw": "新春喜慶掛件組"},
  "描述": {"zh-tw": "農曆新年喜氣洋洋的掛件組合。"},
  "配置": {
    "container-top-left": "圖示:圖示:春聯福字",
    "container-top-right": "圖示:圖示:紅燈籠",
    "page-watermark": "圖示:圖示:祥雲底紋",
    "divider-center-icon": "圖示:圖示:鞭炮"
  },
  "標籤集": ["裝飾:標籤:新年"],
  "售價": 30
}
```

## 7. 佈景主題

### Model (`database/models/佈景主題.ts`)

佈景主題只存指針（ID），指向上述五個模組。同時兼容舊資料欄位名稱。

```typescript
class 佈景主題 {
  名稱: MultilingualString;
  描述: MultilingualString;
  佈景主題: string;   // 自身的完整 ID
  配色: string;        // → "配色:配色:經典藍"
  骨架: string;        // → "骨架:骨架:經典"
  風格: string;        // → "風格:風格:經典實心"
  動畫: string;        // → "動畫:動畫:基本"
  裝飾: string;        // → "裝飾:裝飾:無裝飾"
  售價: number;
}
```

> 相容性：建構子中 `配色` 也會嘗試讀取舊欄位 `配色色庫`，`骨架` 嘗試 `骨架結構`。新增主題時使用新欄位名稱即可。

### 如何建立新的佈景主題

```json
{
  "id": "佈景主題:佈景主題:北歐極簡",
  "名稱": {"en": "Nordic Minimal", "zh-tw": "北歐極簡"},
  "描述": {"zh-tw": "乾淨、溫暖、大量留白的北歐風格。"},
  "配色": "配色:配色:簡約白",
  "骨架": "骨架:骨架:寬鬆舒適",
  "風格": "風格:風格:毛玻璃",
  "動畫": "動畫:動畫:彈跳風",
  "裝飾": "裝飾:裝飾:無裝飾",
  "標籤集": ["佈景主題:標籤:北歐風", "佈景主題:標籤:簡約風"],
  "售價": 0,
  "可刪除": false
}
```

## 8. 工作流程：與 AI 討論建立新佈景主題

當使用者說「我想要一個溫暖的秋天風格主題」時，AI 應按以下步驟引導：

### Step 1：確認配色

請使用者描述想要的顏色感覺，或直接給 OKLCH 值（推薦）。AI 根據描述產出 9 色定義後寫入 `配色.json`。

### Step 2：確認風格

從既有風格中選擇（`經典實心` / `外框線`），或基於既有風格修改 CSS 組合拳來新建。

### Step 3：確認骨架

通常是沿用 `骨架:骨架:經典` 或微調間距/圓角 Token。

### Step 4：確認動畫

沿用 `動畫:動畫:基本` 或新建動畫組合。

### Step 5：確認裝飾

沿用 `裝飾:裝飾:無裝飾` 或根據節日/主題新建裝飾組。

### Step 6：組裝佈景主題

在 `佈景主題.json` 中新增一筆，指向上面五個 ID。

---

## 9. 不要做的事

- **不要** 把配色/風格/骨架的值內嵌到佈景主題裡 — 一律用 ID 指向
- **不要** 為了新元件去改其他模組的程式碼 — `配置` 欄位是 open-ended map，加 key 就好
- **不要** 刪除已在使用的模組 — 佈景主題存的是指針，刪除會造成懸空引用
- **不要** 在 frontend 直接 hardcode 顏色值 — 一律透過 OKLCH CSS 變數讀取
