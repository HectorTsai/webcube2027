# 佈景主題系統設計

## 概述
WebCube2027 的 AI 驅動佈景主題系統，支援自動生成配色、骨架，以及中文命名元件。

## 系統架構

### 佈局層級
```
系統層 (KV/SurrealDB webcube namespace)
├── 配色 (Colors)
├── 骨架 (Structure)  
├── 樣板 (Templates)
└── 元件庫 (Components)

網站層 (各租戶資料庫)
├── 網站配色
├── 網站骨架
├── 自訂元件
└── 內容樣板
```

## Component 架構設計

### 混合載入策略
WebCube2027 支援兩種元件儲存方式，提供最大的靈活性：

#### 1. 檔案系統元件 (開發階段)
- **位置**：`src/components/` 資料夾
- **風格分類**：`實心/`、`線框/`、`幽靈/` 子資料夾
- **載入方式**：直接 import，型別安全
- **優勢**：開發便利、Git 版本控制、編譯時檢查

#### 2. 資料庫元件 (營運階段)
- **模型**：`database/models/元件.ts`
- **儲存**：JSX/TSX 程式碼字串
- **載入方式**：動態執行
- **優勢**：動態管理、多租戶支援、版本控制、熱更新

### 載入優先順序
```typescript
async function 載入元件(元件名稱: string, 風格: string) {
  try {
    // 1. 優先從檔案系統載入
    const 檔案路徑 = `components/${風格}/${元件名稱}.tsx`
    return await import(檔案路徑)
  } catch (錯誤) {
    // 2. 檔案不存在，從資料庫載入
    console.log(`[元件載入] 檔案不存在，從資料庫載入: ${元件名稱}`)
    return await 從資料庫載入元件(元件名稱, 風格)
  }
}
```

### 元件路由器設計
```typescript
// src/components/按鈕.tsx
export default async function 按鈕路由器(props: any, ctx?: Context) {
  const 風格 = ctx?.get('骨架資料')?.風格 || '實心'
  
  try {
    // 嘗試從檔案系統載入
    const 元件模組 = await import(`./${風格}/按鈕.tsx`)
    const 元件 = 元件模組.default
    return 元件(props)
  } catch (錯誤) {
    // 檔案不存在，從資料庫載入
    const 資料庫元件 = await ctx?.get('元件管理器')?.取得元件('按鈕', 風格)
    if (資料庫元件) {
      return 資料庫元件(props)
    }
    
    // 都找不到，使用預設元件
    const 預設模組 = await import('./實心/按鈕.tsx')
    return 預設模組.default(props)
  }
}
```

### 元件管理器
```typescript
// src/services/元件管理器.ts
export class 元件管理器 {
  constructor(private tenantDB: Surreal資料庫) {}
  
  async 取得元件(元件名稱: string, 風格: string): Promise<Function | null> {
    const 查詢 = `
      SELECT 代碼 FROM 元件 
      WHERE 名稱 = $元件名稱 
      AND 風格 = $風格 
      AND 審核通過 = true
      LIMIT 1
    `
    
    const 結果 = await this.tenantDB.查詢(查詢, {
      元件名稱,
      風格
    })
    
    if (結果[0]?.代碼) {
      return this.執行元件程式碼(結果[0].代碼)
    }
    
    return null
  }
  
  private async 執行元件程式碼(代碼: string): Promise<Function> {
    // 安全執行資料庫中的元件程式碼
    return new Function('return ' + 代碼)()
  }
}
```

### 風格分類架構
```
components/
├── 按鈕.tsx (路由元件 - 混合載入)
├── 實心/
│   └── Button.tsx
├── 線框/
│   └── Button.tsx
└── 幽靈/
    └── Button.tsx
```

### 參數設計原則
- **必要英文參數**：`children`, `className`, `onClick`, `type` 等
- **中文參數**：`顏色` (主要、次要、成功等), `尺寸` (小、中、大), `禁用`
- **風格控制**：由 `骨架.風格` 全域控制，不在元件參數中指定

### 動態載入機制
- 路由元件根據 `骨架.風格` 動態載入對應風格的實作
- 支援擴展：可輕鬆新增 `漸層/`、`動畫/`、`異形/` 等風格
- 回退機制：載入失敗時回退到預設風格

### 優勢分析

#### 混合模式的好處
- **向後相容**：現有檔案系統元件完全不受影響
- **漸進增強**：新功能是可選的增強
- **開發便利**：開發階段使用檔案系統，營運階段使用資料庫
- **降級策略**：檔案系統 → 資料庫 → 預設元件

#### 資料庫元件的應用場景
- **網站客製化**：為特定客戶開發專屬元件
- **元件市場**：預設審核通過的元件庫
- **A/B 測試**：動態切換不同版本的元件
- **權限控制**：根據使用者等級提供不同元件
### 回退機制
```typescript
// 載入失敗時回退到預設風格
async function 安全載入元件(元件名稱: string, 風格: string) {
  try {
    return await 動態載入元件(元件名稱, 風格)
  } catch {
    return await 動態載入元件(元件名稱, '實心')
  }
}
```

## AI 生成機制

### 生成流程
```
使用者輸入風格描述
    ↓
AI 分析並提取關鍵詞
    ↓
生成配色方案 + 骨架設定
    ↓
產出 CSS Variable
    ↓
儲存為佈景主題
```

### 輸入範例
```typescript
// 使用者輸入
const 使用者輸入 = "我想要一個溫暖、現代、簡約的風格，像日出一樣的感覺"

// AI 分析結果
const 分析結果 = {
  情感: '溫暖',
  時代: '現代', 
  複雜度: '簡約',
  意象: '日出'
}

// 生成配色
const 生成配色 = {
  primary: '#f59e0b',    // 日出橙
  secondary: '#fb923c',  // 淺橙
  accent: '#fbbf24',     // 日黃
  neutral: '#78350f',    // 深棕
  base: '#fef3c7'        // 溫暖白
}
```

### AI 模型整合
```typescript
import { SmartMultilingual } from '@dui/smartmultilingual'

class 佈景主題生成器 {
  constructor(private ai: SmartMultilingual) {}
  
  async 生成佈景主題(描述: string): Promise<佈景主題> {
    const 分析 = await this.ai.分析風格(描述)
    const 配色 = await this.ai.生成配色(分析)
    const 骨架 = await this.ai.生成骨架(分析)
    
    return {
      名稱: 分析.主題名稱,
      配色,
      骨架,
      描述: 分析.描述
    }
  }
}
```
