# Project Specification: AI-Native Multi-tenant Website Generate (Deno + Hono)

## 1. 專案概述 (Project Overview)
建立一個基於 Deno 的高度動態、多租戶網站系統。
- **核心理念**：全資料庫驅動 (Database-driven)。路由、頁面、樣板均不依賴實體檔案，而是從資料庫讀取並動態渲染。
- **AI 介入**：除了系統預設組件外，頁面/內容/方塊(佈局樣板由) 皆由 AI 生成並以 JSON 或 HonoJSX 字串格式儲存。
- **解耦設計**：渲染層不直接操作 DB，完全透過內部調用 (`app.request` 或 `InnerAPI(c, path)`，傳入 Context 可節省資源) 向 API 索取資料。

## 2. 技術棧 (Tech Stack)
- **Runtime**: Deno (使用最新穩定版)
- **Framework**: Hono (用於 API 與 JSX 伺服器端渲染)
- **Databases (三層架構)**:
    1. **L1 (SQLite)**: 儲存全域種子設定與系統資訊。
    2. **L2 (System SurrealDB)**: 儲存全網站共用資料（字典、圖示、佈景主題等），以及網站設定（名稱、Host、描述、logo）。
    3. **L3 (Tenant Any DB)**: 個別網站的私有內容與設定，支援 SurrealDB / SQLite / MongoDB 多種後端。
    - **資料入口**：`資料池` 全域單例（`database/資料池.ts`），所有服務直接 `import { 資料池 }` 即可操作資料。
    - **路由規則**：查詢 L3（指定 host）→ L2（SYSTEM）→ L1（BASE）純降級；寫入不降級（host→L3、無 host→L2→L1）。
    - **資料庫 ID 規則**：格式為 `table:model:id`（例：`系統資訊:系統資訊:預設`）
    - **連線資訊管理**：L2 連線資訊存於 L1；L3 連線資訊存於 L2。不需要 `.env` 配置。
- **環境變數 (.env)**:
    - `SECRET_KEY`: 密碼加密金鑰。
    - `CACHE_ENABLED`: 快取開關（`true`/`false`），開發時建議關閉。
    - `AI_API_KEY` / `AI_PROVIDER`: LLM 配置（如 `openai` / `anthropic`）。
    - `LOG_LEVEL`: 日誌等級（`debug` / `info` / `warn` / `error`），預設 `info`。
    - `LOG_TO_FILE`: 是否寫入檔案，預設 `false`。
- **Styling**: 
    - UnoCSS (Runtime 模式)，`@unocss/preset-wind4`。
    - 自訂 preset（學習 daisyUI 設計理念），CSS Variables + oklch 色彩。
    - 顏色亮度 ≥40% 用黑色文字，<40% 用白色文字。
    - 提供 API 讓 AI 取得自訂 classes 列表。

## 3. 核心架構規範 (Architectural Rules)

### A. 資料存取層 (API-First)
- 禁止在 Renderer 中直接引用資料庫 Client。所有回傳必須是 JSON 格式。
- 所有 CRUD 必須定義 `/api/v1/*` 路由。
- 渲染層使用 `app.request()` 或 `InnerAPI(c, path)` 獲取數據（詳見 §3.E）。
- **共通欄位**：所有 Model 皆包含 `可刪除: boolean`。
    - Seed 資料 `可刪除 = false`，不可刪除。
    - DELETE API 需檢查此欄位，`false` 時回傳 `403 DELETE_PROTECTED`。

### B. 統一資料庫訪問架構 (Data Pool Architecture)

**✅ 唯一允許的資料訪問方式**：
- **資料池** (`database/資料池.ts`)：全域單例，所有資料操作必須透過此入口。
  - `查詢單一<T>(id, host)` / `查詢列表<T>(model, limit, offset, host)` / `查詢所有列表<T>(model, limit, offset, host)` / `創建或更新<T>(model, data, host)` / `刪除(id, host)` / `取得預設值<T>(model)`
  - `System` getter：直接拿 L2 DatabaseAdapter，供後台系統設定使用
  - host 參數來自 middleware 解析的 request host（含埠號），決定寫入/查詢目標層級
- **Context 預載資訊**（由 Middleware 預載）：
  - `c.get('host')` / `c.get('系統資訊')` / `c.get('網站資訊')`
- **資料過濾器** (`utils/資料過濾器.ts`)：`一般過濾(data, language)` / `列表過濾(data, language, mode)`

**路由邏輯**：
- **查詢**：L3（指定 host）→ 其他 L3 → L2（SYSTEM）→ L1（BASE）純降級，找到資料就回
- **寫入/刪除**：不降級，只寫目標層級（host→L3、無 host→L2→L1）
- **查詢所有列表**：合併所有層級資料，同 id 去重（高層級優先）
- **取得預設值**：永遠只查 L1 seed

**生命週期**：
- `main.ts` → `await 資料池.初始化()` → L1 連線 + 動態掃描 `database/models/*.ts` 自動 seed
- middleware → `await 資料池.初始化L2()` → L2 連線
- middleware → `await 資料池.初始化L3(host)` → L3 連線（以 host 為 key）

**分層權限表**：

| 層級 | ✅ 允許 | ❌ 禁止 |
|------|--------|--------|
| **Middleware** | Context 操作、資料池 | 無 |
| **API 層** | Context 取得、資料池 | 直接資料庫連線 |
| **Service 層** | Context 取得、資料池 | 直接資料庫連線 |
| **Renderer 層** | API 調用 (`app.request` / `InnerAPI(c, path)`)，可傳入 Context 給 InnerAPI | Context 直接操作、Service、直接資料庫 |

**關鍵規則**：
- ✅ 所有服務層級都可使用資料池（不再需要透過 Context 拿 DB）
- ✅ Renderer 層只能透過 API 調用獲取資料
- ⚠️ AI 編輯器必須遵循此架構限制

### C. 渲染管線 (Rendering Pipeline)
- **路由分發**：`app.get('*')` 動態分發，依序攔截：
    - `/api/*` → apiService（回傳 JSON 錯誤）
    - `/medias/*` → mediaService（回傳預設資源）
    - 其餘 → rendererService（回傳 404 頁面）
    - 各 Service 自行處理錯誤及 404。
- **CSS 處理**：渲染 HTML 後呼叫 `unocss.generate(html)`，注入 `<head><style>`。
- **佈景主題系統**：`佈景主題 = 骨架 + 配色 + 風格 + 動畫 + 裝飾`，五個模組各自獨立存在資料庫，佈景主題只儲存五個指針（ID），可跨主題共用模組。
    - **骨架**：幾何結構、字型尺寸、間距、圓角、佈局方塊 ID。
    - **配色**：OKLCH 色彩定義（主色/次色/強調/中性/背景/功能色，共 9 色）。
    - **風格**：CSS class 組合拳（active/inactive/hover 三態 + 漸層 + 陰影）。
    - **動畫**：UI 互動動畫 class 映射（抽屜/視窗/下拉/彈出）。
    - **裝飾**：頁面掛件與 SVG 裝飾（角落徽章、浮水印）。
    - 所有模組的 `配置` 欄位為 open-ended map，擴充只需加 key，不需改程式碼。
    - 詳見 [佈景主題系統設計指南](./佈景主題系統設計指南.md)。
- **方塊系統**：資料驅動的 UI 元件系統，每個方塊是一筆 JSON（含 `from`、`className`、`style`、`args`、`children`、`mergedArgs` 等），`Cube.tsx` 是萬用渲染器。
    - `from: "div"|"span"|...` → 原生 HTML，Cube.tsx 直接渲染，支援 `text` 屬性設定文字內容。
    - `from: "方塊:方塊:..."` → 方塊 ID，透過 fallback registry 委派給對應元件（如 Container.tsx）。
    - 支援 JSX children（靜態頁面）和資料驅動（seed/API 回傳）兩種等價用法。
    - 型別使用 `Partial<方塊>`，children 支援物件或字串，遞迴深度上限 10 層。

    **動態資料機制**（兩種模式可混用）：
    - **顯式 `$api`**：在 children / slots 的屬性值中使用 `{ "$api": "欄位名" }`（預設查 `/api/v1/info`）或 `{ "$api": { "path": "/api/v1/...", "field": "欄位名" } }`（自訂 API 路徑）。path/field 支援 `{variable}` 插值（如 `"/api/v1/page/{item}"`），搭配 `repeat` 使用。方塊渲染時自動呼叫對應 API 取得資料並替換。
    - **隱式 `mergedArgs`**：在方塊定義頂層宣告 `"mergedArgs": { "變數名": "@api/v1/..." }`，解析後自動注入 `mergedArgs`，並隱式傳遞給所有子方塊（`renderCubeChildren` 自動繼承父層參數）。適合多個子方塊共用的資料。
    - **`repeat` 展開**：在 child 定義中加入 `"repeat": "{items}"`，會從 `mergedArgs` 取出 `items` 陣列，為每個元素建立一個方塊實例，並將元素值設為 `{item}` 變數供 `$api` 插值使用。
    - **slots.children**：slots 內的每個 slot 現在可直接定義 `children`（含 `repeat`/`$api`），renderer 會自動將它們作為 slot 預設值渲染。若有外部 `<Slot>` 傳入，則覆蓋預設值。
    - 兩者皆透過 `InnerAPI(c, path)` 內部調用，共享請求快取。
    - `className` / `style` / `containerClassName` 在 JSON 子元素中會穿透到 fallback 元件。

    - 詳見 [方塊設計指南](./方塊設計指南.md)。

### D. 多國語言 (MultiLanguage)
- 使用 `jsr:@dui/smartmultilingual@0.2.2`，語言服務統一由 `services/languageService` 管理。
- **語言解析優先序**：Cookie (`lang`) → Accept-Language 標頭 → 統一資訊 API → 預設 `zh-tw`。
- **API 語言使用**：`c.set('語言', language)` → `c.get('語言')`，多國語言字串用 `toStringAsync(language)`。
- Model 使用 `MultilingualString` / `SmartContent` 儲存多國語言內容。

### E. 內部 API 調用規範 (InnerAPI)
- **強制使用 InnerAPI**：所有非 Renderer 層的內部 API 調用必須使用 `services/index.ts` 的 `InnerAPI(c, path)`。
  - 自動編碼 URL 參數、統一錯誤處理、自動附加必要 HTTP 標頭（host、origin、cookie）。
  - **禁止**直接使用 `app.request()`。
- **Renderer 層**可使用 `app.request()` 或 `InnerAPI(c, path)`（傳入 Context 節省資源）。
- **實作細節**（參數解碼、智能路由、檔案命名等）請參考 `docs/開發流程書.md`。

### F. Media Service 規格
- **路由攔截**：`/medias/*` 由 mediaService 處理。
- **路徑格式**：列表 `/medias/{type}`（支援 `page`, `pageSize`, `sort`, `order`, `search`），單筆 `/medias/{type}/:id`。
- **特殊路徑**：`/medias/icon` → 網站/系統資訊的商標，無則回傳預設圖示（favicon）。
- **找不到時回傳對應類型的預設資源**（非 JSON 錯誤）：圖示→預設 SVG、圖片→placeholder、影片→封面。
- SVG 圖示存於 SurrealDB，其他媒體儲存方式待定。

### G. API 回應格式標準
- **成功回應**：`{ success: true, data, source }`
- **錯誤回應**：`{ success: false, error: { code, message, status } }`
- **HTTP Status Code**：200 查詢成功 / 201 新增成功 / 400 請求格式錯誤 / 403 禁止操作 / 404 資源不存在 / 422 驗證失敗 / 500 內部錯誤
- **錯誤 Code 常數**：`NOT_FOUND` / `FORBIDDEN` / `DELETE_PROTECTED` / `VALIDATION_ERROR` / `BAD_REQUEST` / `INTERNAL_ERROR`

### H. SEO 與網站資訊
- 網頁 SEO 資料（title、description、og:image）從網站資訊取得，若無則回退到系統資訊（L1）。
- 網站資訊 Model 包含：網站名稱、描述、商標、社群圖片等。

### I. 快取策略 (Caching)
- 透過 `.env` 的 `CACHE_ENABLED` 控制開關，開發環境預設關閉。

### J. 命名規則 (Naming Convention)
- **繁體中文為主**：對外變數、函數、類別、Model 欄位、API JSON key、Seed 欄位。
    - 例：`佈景主題`、`可刪除`、`取得頁面()`
- **內部可用英文**：區域變數、輔助函數。
    - 例：`const res = ...`、`let i = 0`
- 第三方套件相關使用英文（`app.get()`、`c.json()`）。

### K. RESTful API 總覽
> **此章節為 API 的唯一權威定義，所有其他章節均參考此處。**

**設計原則**：全部單數命名檔案（`theme.ts`、`color.ts`），使用路徑參數區分操作（`/{resource}/all`、`/{resource}/{id}`），舊版 query 參數逐步淘汰。

#### K.1 核心 CRUD API（六模組）

每個資源支援以下模式：

| 方法 | 路徑 | 用途 |
|------|------|------|
| `GET` | `/api/v1/{resource}` | 取得當前資源 |
| `GET` | `/api/v1/{resource}/all` | 取得所有（支援 `?include_seeds=false`） |
| `GET` | `/api/v1/{resource}/{id}` | 取得指定資源 |
| `POST` | `/api/v1/{resource}` | 建立新資源 |
| `PUT` | `/api/v1/{resource}/{id}` | 更新資源 |
| `DELETE` | `/api/v1/{resource}/{id}` | 刪除（檢查 `可刪除`） |

適用資源：`theme`、`color`、`skeleton`、`style`、`animation`、`ornament`

**三層查詢邏輯**：L3 → L2 → L1 (SQLite seeds)，保證預設值存在。系統 seeds 標示 `isSystemSeed: true` 且 `source: "L1"`，刪除時回傳 `SYSTEM_SEED_PROTECTED`。

#### K.2 頁面與方塊 API

- **page**：`GET /api/v1/page/all`、`GET /api/v1/page/{id}`、`GET /api/v1/page/{id}/標題`（巢狀欄位，類似 `/api/v1/info` 模式）、`GET /api/v1/page/path/{path}`（巢狀路徑）、`POST`、`PUT`、`DELETE`
- **cube**：`GET /api/v1/cube`（全部）、`GET /api/v1/cube/{id}`、`POST`、`PUT`、`DELETE`

#### K.3 統一資訊 API

| 端點 | 用途 |
|------|------|
| `GET /api/v1/info` | 統一資訊（優先網站 → 回退系統），含語言列表、當前五模組設定 |
| `GET /api/v1/info/system` | 系統資訊 |
| `GET /api/v1/info/website` | 網站資訊 |
| `GET /api/v1/info/{巢狀欄位路徑}` | 巢狀欄位存取（如 `/info/版權資料/公司`），逐層取值，不存在回 404 |

#### K.4 預設值 API

`GET /api/v1/defaults` / `GET /api/v1/defaults/{module}` — 取得各模組預設值（L3→L2→L1 回退）。

#### K.5 UnoCSS API

| 端點 | 用途 |
|------|------|
| `GET /api/v1/unocss` | 所有自訂 classes |
| `GET /api/v1/unocss/classes` | AI 可用 class 列表 |
| `GET /api/v1/unocss/info` | 主題資訊 |

#### K.6 回應格式

```json
{
  "success": true,
  "data": { ... },
  "source": "L1" | "L2" | "L3",
  "pagination": { "limit": 10, "offset": 0, "total": 100 }
}
```

#### K.7 日期時間 API

| 端點 | 用途 |
|------|------|
| `GET /api/v1/temporal` | 本地化日期字串（依 `c.get('語言')`） |
| `GET /api/v1/temporal/fullYear` | 當前年份 |
| `GET /api/v1/temporal/month` | 當前月份 (1-12) |
| `GET /api/v1/temporal/day` | 當前日期 (1-31) |
| `GET /api/v1/temporal/hours` | 當前小時 (0-23) |
| `GET /api/v1/temporal/minutes` | 當前分鐘 (0-59) |
| `GET /api/v1/temporal/seconds` | 當前秒數 (0-59) |

基於 Deno 原生 Temporal API，查不到欄位回 404。

#### K.8 翻譯 API

| 端點 | 用途 |
|------|------|
| `GET /api/v1/translate/{key}` | 將英文 key 翻譯為當前使用者語言 |

翻譯管線：AI 翻譯（via AI Pool）→ Google 翻譯 fallback → 寫入快取（單字 model）。後續請求命中快取直回，不再調用外部服務。

### L. AI 方塊生成規範 (AI Cube Generation)

#### L.1 生成 API
- `POST /api/v1/ai/generate-cube`：`{ "描述": "...", "目標": "system" | "tenant" }` → 產生方塊並存入 DB。
- 支援多 LLM Provider（Ollama / Cloud API），透過 `database/seeds/AI伺服器.json` 設定。
- 所有 AI 請求由 `services/aiService/pool.ts` AI Pool Manager 統一管理，依任務能力需求自動選擇最適伺服器。

#### L.2 Prompt 管理
- Prompt 不再硬編碼於程式碼中，儲存在 `database/models/AI提示詞.ts`（資料庫管理，可線上修改即時生效）。
- 預設內建 9 組 Prompt（Cube / Page / Style 生成、代碼審查、翻譯、客服等），透過 `database/seeds/AI提示詞.json` 初始化。
- 生成時動態注入 `/api/v1/unocss/classes` 取得的可用 class 列表。
- 強制規則：
  - 僅使用 Hono JSX 語法（非 React JSX）。
  - **禁止** arbitrary values（如 `text-[#ff0000]`）。
  - **禁止** `<script>` 標籤及 inline event handlers。
  - 組件必須接收 `props` 作為資料輸入。

#### L.3 三層安全管線

##### L3.1 L1 — 寫入前正則過濾
- 模組：`utils/安全過濾器.ts`
- 三合一正則過濾：
  - Script 標籤移除
  - `javascript:` URI 攔截
  - inline event handlers（onclick, onload, onerror 等）移除
- 同時計算完整性雜湊（SHA-256）：將方塊關鍵欄位序列化後計算 hash，存入 `已檢驗` 欄位。

##### L3.2 L2 — 排程 AI 審查
- 模組：`services/aiService/task/code-review.ts`
- 使用**獨立 AI Provider**（與代碼生成不同模型），避免審查盲點。
- 由排程系統觸發，掃描 `已檢驗` 為空的方塊進行安全審查。
- 通過 → 計算 hash 寫入 `已檢驗`。
- 失敗 → 標記 `已檢驗: '審查失敗'` + 記錄原因，等待人工處理。

##### L3.3 L3 — 渲染前 hash 驗證
- 在 `components/Cube.tsx` 實作：渲染方塊時重新計算 SHA-256 hash。
- 比對 DB 中 `已檢驗` 欄位的 hash 值。
- 不符 → 顯示安全性警告佔位元件，不渲染原始內容。
- 符合 → 正常渲染。

#### L.4 自動重試機制
- 模組：`utils/AI重試.ts`，通用工具函數 `聊天並解析JSON()`。
- AI 回應後自動嘗試解析 JSON，解析失敗時：
  - 自動追加重試提示後再次請求（可設定最大重試次數）。
  - 降低 temperature（0.7 → 0.3）提高穩定性。
- 已套用於所有 Task：cube-generator、page-generator、style-generator、code-review。

#### L.5 失敗處理
- LLM 呼叫失敗：`500 INTERNAL_ERROR`。
- 產出不合法：`422 VALIDATION_ERROR`，自動重試後仍失敗才回錯誤。
- 渲染失敗：佔位元件 + 日誌記錄。

### M. 日誌系統 (Logging)
- **統一日誌介面**：`utils/logger.ts`，全專案禁止直接使用 `console.log`。
- **日誌等級**（`.env` 的 `LOG_LEVEL`）：`debug`（詳細除錯）/ `info`（一般運行）/ `warn`（警告）/ `error`（錯誤）。
- **輸出**：預設 console，`LOG_TO_FILE=true` 時寫入 `/logs` 目錄。

### N. 認證/授權系統 (Authentication/Authorization)
- **第一階段**：暫不實作，所有 API 開放存取。
- **後續階段**：加入「使用者」與「角色」Model，POST/PUT/DELETE 需認證保護，依角色區分系統管理員/網站管理員權限。

## 4. 目錄結構 (Folder Structure)
- `/unocss`: UnoCSS 核心模組（rules、theme、shortcuts、generator、cache、animate）。
- `/data`: SQLite 資料庫實體檔案。
- `/database`:
    - `資料池.ts`: 全域單例，L1/L2/L3 統一資料入口（路由、降級、寫入、seed 管理）
    - `/adapter`: L3 多資料庫 adapter（SurrealDB、SQLite、MongoDB）。
    - `/models`: Model 定義。
    - `/seeds`: 初始資料。
- `/docs`: 設計指南與規格書。
- `/middleware`: DB 解析器 (db-resolver)、資訊預載 (info-loader)、語言解析 (language-resolver)。
- `/services`:
    - `/apiService`: 處理 `/api/*` 路由，業務邏輯層。
    - `/mediaService`: 處理 `/medias/*` 路由。
    - `/rendererService`: 處理其餘路由，調用 API 並回傳 HTML。
    - `/languageService`: 語言管理與驗證。
- `/components`: 預設方塊組件 (.tsx)。
- `/utils`: Logger、安全過濾 (sanitizer)、AI prompt 引擎。
- `main.ts`: 程式進入點。
