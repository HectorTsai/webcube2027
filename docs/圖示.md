# 圖示系統設計

## 概述

WebCube2027 的圖示系統採用統一設計，支援載入動畫、品牌圖示、使用者頭像等多種用途，提供簡潔且高效的使用體驗。

## 設計原則

### 1. 元件純粹性
- **不直接依賴資料庫**：元件透過 props 接收資料，保持純粹性
- **職責分離**：資料庫邏輯在元件內部處理，外部只需傳入 ID
- **易於測試**：可以輕易傳入不同 ID 進行測試

### 2. 效能優先
- **直接 SVG 渲染**：使用 `dangerouslySetInnerHTML` 避免 React 包裝開銷
- **按需載入**：支援動態載入，不增加初始包大小
- **快取機制**：資料庫查詢結果快取，避免重複請求
## 載入器元件

### 設計決策

```typescript
// src/components/Loading.tsx
interface LoadingProps {
  風格?: 'spinner' | 'dots' | 'pulse' | 'skeleton'
  內容?: string
}

export default async function Loading({ 
  風格 = 'spinner', 
  內容 
}: LoadingProps) {
  try {
    // 1. 如果有傳入自定義 SVG 內容，直接使用
    if (內容 && 內容.includes('<svg')) {
      return <span dangerouslySetInnerHTML={{ __html: 內容 }} />
    }
    
    // 2. 否則嘗試從檔案載入預設風格
    const svgPath = new URL(`./loadings/${風格}.svg`, import.meta.url)
    const svgContent = await Deno.readTextFile(svgPath)
    
    return <span dangerouslySetInnerHTML={{ __html: svgContent }} />
  } catch (_e) {
    // 3. 後備方案：CSS 動畫
    return (
      <span className="inline-block animate-spin">
        <div className="w-4 h-4 border-2 border-slate-700 rounded-full border-t-transparent"></div>
      </span>
    )
  }
}
```

### 為什麼選擇這個設計？
#### 1. 直接 SVG 而非 React 元件
- **效能更好**：避免 React 包裝開銷
- **檔案更小**：不需要額外的 TypeScript 程式碼
- **維護簡單**：只需要維護 SVG 檔案

#### 2. `<span>` 而非 `<div>`
- **inline 佈局**：不會強制換行，可以和文字並排
- **靈活使用**：使用端完全控制佈局
- **語意正確**：載入器通常是內聯元素

#### 3. 三層後備機制
- **自定義 SVG**：最大靈活性
- **檔案系統**：預設風格，快速載入
- **CSS 動畫**：最終保險，確保總有載入器

## 圖示元件設計

### 圖示元件

```typescript
// src/components/圖示.tsx
interface 圖示Props {
  id: string        // 圖示 ID
  size?: number | string  // 尺寸
  className?: string      // CSS 類名
}

export default async function 圖示({ id, size, className }: 圖示Props) {
  // 從資料庫取得圖示
  const icon = await 圖示.取得(id)
  if (!icon) return null
  
  const { 資料, 格式 } = icon.資料
  
  // 根據格式渲染
  switch (格式) {
    case 'SVG':
      return (
        <span 
          className={className}
          style={{ width: size, height: size }}
          dangerouslySetInnerHTML={{ __html: 資料 }} 
        />
      )
      
    case 'PNG':
    case 'ICO':
      return (
        <img 
          src={資料}
          alt={icon.名稱.取得('zh-tw')}
          width={size}
          height={size}
          className={className}
        />
      )
      
    default:
      console.warn(`不支援的圖示格式: ${格式}`)
      return null
  }
}
```

  }
}

## 最佳實踐

### 1. 新增圖示
1. 在 `database/seeds/圖示.json` 加入圖示資料
2. 支援的格式：SVG 內容、PNG/ICO 圖片路徑
3. 使用描述性的 ID（如 `webcube:logo`、`載入器:spinner`）

### 2. 使用圖示元件
```typescript
// 基本使用
<圖示 id="webcube:logo" />

// 自訂尺寸
<圖示 id="載入器:spinner" size={32} />

// 加 CSS 類名
<圖示 id="user:avatar" size={48} className="rounded-full shadow-lg" />
```

### 3. 載入動畫使用
```typescript
// 基本載入動畫
<圖示 id="載入器:spinner" size={24} className="animate-spin" />

// 不同風格的載入動畫
<圖示 id="載入器:dots" size={32} />

// 與文字並排
<span>載入中...<圖示 id="載入器:spinner" size={16} /></span>
```

### 4. 圖示格式規範
- **SVG**: 直接儲存完整的 SVG 代碼，可包含動畫
- **PNG**: 儲存圖片檔案路徑
- **ICO**: 儲存圖示檔案路徑

### 5. 效能考量
- 圖示元件會自動從資料庫載入
- 考慮在生產環境中加入快取機制
- SVG 使用 `dangerouslySetInnerHTML` 確保最佳效能
- 載入動畫使用 CSS 類名控制動畫效果

## 總結

這個圖示系統的設計強調：

- **簡潔性**：單一 `<圖示>` 元件，API 簡單直觀
- **靈活性**：支援 SVG、PNG、ICO 多種格式
- **效能性**：直接渲染，無額外包裝開銷
- **可維護性**：元件職責單一，易於理解和維護
- **可擴展性**：易於新增新格式和功能

透過這個設計，WebCube2027 能提供一致且高效的圖示使用體驗，同時為未來的 AI 圖示生成系統做好準備。
