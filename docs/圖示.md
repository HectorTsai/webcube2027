# 圖示與載入器系統設計

## 概述

WebCube2027 的圖示與載入器系統採用模組化設計，支援檔案系統和資料庫兩種載入方式，提供靈活且高效的使用體驗。

## 設計原則

### 1. 元件純粹性
- **不直接依賴資料庫**：元件透過 props 接收資料，保持純粹性
- **職責分離**：資料庫邏輯在路由層處理，元件只負責渲染
- **易於測試**：可以輕易傳入不同資料進行測試

### 2. 效能優先
- **直接 SVG 渲染**：使用 `dangerouslySetInnerHTML` 避免 React 包裝開銷
- **按需載入**：支援動態載入，不增加初始包大小
- **快取機制**：資料庫查詢結果快取，避免重複請求

### 3. API 簡潔
- **參數分離**：`風格` 用於預設選項，`內容` 用於自定義
- **TypeScript 支援**：完整型別提示，減少使用錯誤
- **統一介面**：圖示和載入器使用一致的 API 設計

## 按鈕元件

### 設計決策

```typescript
// src/components/按鈕.tsx
interface ButtonProps {
  風格: string // 必要參數，從 route 傳入
  // ... 其他屬性
}

export default async function Button({ 風格, ...props }: ButtonProps) {
  try {
    // 1. 嘗試從檔案系統載入
    const Button = (await import(`./${風格}/Button.tsx`)).default
    return <Button {...props} />
  } catch (_e) {
    // 2. 後備方案：使用預設實心按鈕
    return <實心按鈕 {...props} />
  }
}
```

### 為什麼不支援資料庫？

- **按鈕是純 UI 元件**：邏輯簡單，不需要動態內容
- **檔案系統足夠**：預設風格已滿足大部分需求
- **保持簡單**：避免過度設計

### 目錄結構

```

├── 實心/
│   └── Button.tsx     # 實心風格
├── 線框/
│   └── Button.tsx     # 線框風格
└── 幽靈/
    └── Button.tsx     # 幽靈風格
```

## 載入器元件

### 設計決策

```typescript
// src/components/Loading.tsx
interface LoadingProps {
  風格?: 'spinner' | 'dots' | 'pulse' | 'skeleton'
  內容?: string
}

export default async function Loading({ 風格 = 'spinner', 內容 }: LoadingProps) {
  try {
    // 1. 自定義 SVG 內容優先
    if (內容 && 內容.includes('<svg')) {
      return <span dangerouslySetInnerHTML={{ __html: 內容 }} />
    }
    
    // 2. 預設風格從檔案載入
    const svgPath = new URL(`./loadings/${風格}.svg`, import.meta.url)
    const svgContent = await Deno.readTextFile(svgPath)
    
    return <span dangerouslySetInnerHTML={{ __html: svgContent }} />
  } catch (_e) {
    // 3. CSS 動畫後備
    return <span className="inline-block animate-spin">...</span>
  }
}
```

### 為什麼選擇這個設計？

#### 1. 直接 SVG 而非 React 元件
- **效能更好**：避免 React 包裝開銷
- **檔案更小**：不需要額外的 TypeScript 程式碼
- **維護簡單**：只需要維護 SVG 檔案

#### 2. `<span>` 而非 `<div>`
- **inline 佈局**：不會強制換行，可以和文字並排
- **靈活使用**：使用端完全控制佈局
- **語意正確**：載入器通常是內聯元素

#### 3. 三層後備機制
- **自定義 SVG**：最大靈活性
- **檔案系統**：預設風格，快速載入
- **CSS 動畫**：最終保險，確保總有載入器

### 使用場景

```typescript
// 1. 預設風格
<Loading 風格="spinner" />
<Loading 風格="dots" />

// 2. 自定義 SVG
<Loading 內容="<svg class='w-4 h-4'>...</svg>" />

// 3. 與文字並排
<span>載入中...<Loading 風格="spinner" /></span>
```

```
├── dots.svg       # 點點載入器
├── pulse.svg      # 脈衝載入器
```


## 技術決策記錄

### svgson 的取捨
- **最終決定**：不使用 svgson
- **原因**：直接傳入 SVG 字串更簡單
- **保留可能**：未來需要動態修改 SVG 時再考慮

### React 元件 vs 純 SVG
- **最終決定**：使用純 SVG + dangerouslySetInnerHTML
- **原因**：效能更好，維護更簡單
- **例外情況**：複雜的互動式元件仍用 React

### 資料庫依賴
- **按鈕元件**：不依賴資料庫，保持簡單
- **載入器元件**：預留資料庫介面，但不強制使用
