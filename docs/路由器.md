# WebCube2027 檔案路由器

## 概述
WebCube2027 檔案路由器是一個基於檔案系統的自動路由系統，支援 Hono 框架。

## 核心功能
1. **自動掃描** `src/routes` 目錄下的 `.tsx` 檔案
2. **路由映射**：檔案路徑 → URL 路徑
3. **特殊檔案支援**：
   - `_layout.tsx` - 佈局包裝器
   - `_middleware.tsx` - 中間件
   - `_app.tsx` - 全域 App 包裝器
   - `_404.tsx` - 404 錯誤頁面
   - `_error.tsx` - 錯誤處理器
   - `_500.tsx` - 500 伺服器錯誤頁面

## 內部重導向功能

### 設計理念
- **開發者主導**：提供兩種重導向方法，讓開發者根據情境選擇
- **高效能優先**：預設使用內部重導向，避免重新 request
- **安全閥機制**：循環時自動降級為外部重導向

### API 設計
```typescript
// Context 擴展
declare module 'hono' {
  interface Context {
    /** 內部重導向（高效能，避免重新 request） */
    internalRedirect(path: string): Promise<Response>
    /** 重導向鏈追蹤（防止循環） */
    _redirectChain?: string[]
  }
}
```

### 使用範例
```typescript
// _middleware.tsx 或路由組件中
export default async function AuthMiddleware(ctx: Context, next: Next) {
  const user = await getCurrentUser(ctx)
  
  if (!user) {
    // 內部重導向：高效能，適合權限檢查
    return await ctx.internalRedirect('/login')
  }
  
  if (!user.emailVerified) {
    // 外部重導向：需要完整頁面重新載入
    return ctx.redirect('/verify-email')
  }
  
  await next()
}
```

### 自動降級機制
```typescript
// 當檢測到循環時，自動降級為外部重導向
if (ctx._redirectChain.includes(targetPath)) {
  console.warn('檢測到循環，降級為外部重導向')
  return ctx.redirect(targetPath)
}
```

### 效能優勢
- **99% 情況**：使用內部重導向，無新 request
- **避免重複執行**：middleware 鏈不會重新執行
- **減少資料庫查詢**：權限檢查等操作不會重複

### 安全特性
- **循環檢測**：自動檢測重導向循環
- **自動降級**：循環時降級為標準 302
- **路由驗證**：確保目標路由存在
- **除錯日誌**：記錄重導向過程

## 路由規則
- `index.tsx` → `/`
- `about.tsx` → `/about`
- `users/[id].tsx` → `/users/:id`
- `blog/posts/[slug].tsx` → `/blog/posts/:slug`

## 嵌套結構
- 支援巢狀特殊檔案（如 `/blog/_layout.tsx` 只影響 blog 路由）
- 中間件從根到目標路徑順序執行
- Layout 從外到內包裝

## 錯誤處理流程
1. **404 錯誤**：路由找不到時使用最近的 `_404.tsx`
2. **500 錯誤**：伺服器錯誤時使用最近的 `_500.tsx`
3. **一般錯誤**：其他錯誤使用最近的 `_error.tsx`

## 設計原則
1. **單一檔案**：所有路由邏輯在一個檔案中
2. **直接映射**：避免複雜的中間層
3. **Hono 整合**：直接返回 Hono Response
4. **錯誤處理**：簡單明瞭的錯誤流程

## 使用方式
```typescript
import { setupFileRouter } from './src/router.ts'
import { Hono } from 'hono'

const app = new Hono()
await setupFileRouter(app, {
  routesDir: 'src/routes'
})
```

## 特殊檔案類型

### _layout.tsx
```typescript
export default function Layout(Component: () => JSX.Element, ctx: Context) {
  return (
    <div className="layout">
      <header>Header</header>
      <main>{Component()}</main>
      <footer>Footer</footer>
    </div>
  )
}
```

### _middleware.tsx
```typescript
export default function middleware(ctx: Context, next: () => Promise<void>) {
  console.log('Request:', ctx.req.path)
  await next()
}
```

### _app.tsx
```typescript
export default function App(Component: () => JSX.Element, ctx: Context) {
  return (
    <html>
      <head><title>My App</title></head>
      <body>{Component()}</body>
    </html>
  )
}
```

### _404.tsx
```typescript
export default function NotFound404(ctx: Context) {
  return ctx.html('<h1>404 Not Found</h1>', 404)
}
```

### _error.tsx
```typescript
export default function ErrorHandler(error: Error, ctx: Context) {
  return ctx.html(`<h1>Error: ${error.message}</h1>`, 500)
}
```

### _500.tsx
```typescript
export default function ServerError500(ctx: Context, error?: Error) {
  return ctx.html('<h1>Internal Server Error</h1>', 500)
}
```
