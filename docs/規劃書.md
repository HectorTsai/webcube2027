## 目標
- 這是一個網站生成器，可以生成多個網站（以 domain 區分），每個網站都可以獨立運行。
- 系統：deno 2.x
- Framework: Hono
- 資料庫：系統資料庫用 KV，站點資料用 SurrealDB（namespace 區隔）。
- 路由器：自行設計檔案路由器（請盡量設計成容易抽離成獨立套件）
- 媒體：MinIO 儲存圖片/影片/audio（不含 SVG）。
- AI：預設本地 ollama，語言處理用 jsr:@dui/smartmultilingual。
- UI：AI 參照佈景主題（配色+骨架）生成，搭配 unoCSS；並實作書本元件（以翻書模式瀏覽網頁）
- 發佈：docker。
- 文件：Markdown 為主。
- 資料查詢：自然語言轉 AI 查詢。

## 多租戶/站點隔離
- 網站識別：`domain` 為站點 ID。
- KV：僅存系統資訊（系統設定）。
- 系統資料庫：SurrealDB `webcube` namespace，存共享系統資料（網站資訊、佈景主題、骨架、配色、單字(系統用)、語言、標籤(系統用)、圖示）。
- 網站資料庫：根據網站資訊設定，連線到各網站自訂 SurrealDB 實例，存站點專用資料（內容、頁面、單字、標籤、媒體、圖示）。
- 權限：以 `domain` 控制站點資料與媒體存取範圍。

## 資料模型（無 Schema）
- SurrealDB 不建立 schema；在應用層以 class 定義欄位、驗證與預設值（支援中文命名）。
- 建議欄位：`_id`、`_createdAt`、`_updatedAt`、`_createdBy`、`_updatedBy`、`_traceId`。
- AI 查詢白名單
- 範例（中文 class）：
```ts
class 單字 {
  資料:MultilingualString;
  更新時間: Date = new Date();
}
```

## Component 架構設計 (2026-03-10)
### **風格分類架構**
```
components/
├── 按鈕.tsx (路由元件 - 動態載入)
├── 實心/
│   └── Button.tsx
├── 線框/
│   └── Button.tsx
└── 幽靈/
    └── Button.tsx
```

### **參數設計原則**
- **必要英文參數**: `children`, `className`, `onClick`, `type` 等
- **中文參數**: `顏色` (主要、次要、成功等), `尺寸` (小、中、大), `禁用`
- **風格控制**: 由 `骨架.風格` 全域控制，不在元件參數中指定

### **動態載入機制**
- 路由元件根據 `骨架.風格` 動態載入對應風格的實作
- 支援擴展：可輕鬆新增 `漸層/`、`動畫/`、`異形/` 等風格
- 回退機制：載入失敗時回退到預設風格

## Middleware 資料處理 (2026-03-10)
### **語言處理優先順序**
1. **URL 路徑** - `/en/index` → `en`
2. **Cookie 記憶** - `webcube-lang` Cookie (1年過期)
3. **瀏覽器設定** - `accept-language` header
4. **預設語言** - `zh-tw`

### **預設資料策略**
- **初始部署**: 硬編碼預設骨架 + 配色 (給系統管理者第一次使用)
- **翻譯策略**: 預設 + Google 翻譯 (@dui/smartmultilingual)
- **漸進增強**: 管理者可逐步替換為專業資料

### **資料載入優先順序**
```typescript
// 有網站資訊時的載入順序
1. siteInfo.骨架ID (優先)
2. siteInfo.配色ID (優先)
3. 佈景主題.骨架ID (補載)
4. 佈景主題.配色ID (補載)
```

### **State 設定**
```typescript
ctx.set('骨架資料', 骨架資料);
ctx.set('配色資料', 配色資料);
ctx.set('佈景主題資料', 佈景主題資料);
ctx.set('語言', lang);
```

## 媒體庫（MinIO）
- Bucket（少量固定）：
  - `media`：原始媒體。
  - `thumb`：縮圖。
- 物件 key 結構：
  - 原檔：`{domain}/{type}/{yyyy}/{mm}/{dd}/{uuid}.{ext}`
  - 縮圖：`{domain}/thumb/{sizePreset}/{yyyy}/{mm}/{dd}/{uuid}.webp`
  - `type`：`image|video|audio|file`
- Metadata 建議：`content-type`、`original-name`、`sha256`、`width`、`height`、`duration`（影音）。
- 存取：內部以服務帳號並限制 prefix；對外用 pre-signed URL（短時效）。未來可掛 CDN。

## 媒體處理
- 需求：上傳後轉一張縮圖（建議 WebP）。
- 流程：上傳 → 觸發任務（queue/async）→ 轉換 → 存 `thumb` bucket。
- 尺寸：
  - 單尺寸：`thumb-800`（寬 800px，等比縮放）。
  - 可預留多尺寸：`thumb-sm` 400px、`thumb-lg` 1200px。
- 資料庫紀錄：原檔 URL、縮圖 URL、尺寸資訊。

## 媒體支援
- 支援的格式：
  - 圖片：`jpg`, `jpeg`, `png`, `gif`, `webp`, `svg`, `json` (Lottie 動畫)
  - 影音：`mp4`, `webm`, `ogg`, `mp3`, `wav`, `ogg`, `avi`
  - 文件：`pdf`, `doc`, `docx`, `xls`, `xlsx`, `ppt`, `pptx`
- `svg`, `json`，儲存於 SurrealDB

## UI/佈景/元件
- 佈景：系統層（KV/SurrealDB `webcube` namespace）存放配色、骨架、樣板。學daisyui使用CSS variable記錄顏色、圓角等，分類為配色（colors）及骨架（structure，如spacing、border-radius）。
- 元件來源：
  - 可存 DB（方便版本/審核），或檔案庫；AI 生成時僅允許白名單元件與 unoCSS 原子類。
- 生成流程：AI 讀取佈景/元件 → 產出 UI（中文命名 class/變數可用）→ 透過 unoCSS 實體化。
- AI生成佈景主題：除了預設配色/骨架seeds，網站管理者輸入風格、顏色或感受，AI自動產生新佈景主題數值。若滿意，存入資料庫。
- 可存取性：保持語義化、鍵盤可用、WCAG 基礎。

## 路由與授權
- Domain 綁定。
- 權限：站點擁有者/編輯/訪客；API/內部服務需 RBAC 與速率限制（後續補）。
- Router路徑：
  - `/index.tsx`：首頁。
  - `/system/...`：系統相關。
  - `/web/...`：網站管理相關。
  - `/api`：API 入口。
  - `/api/system/...`：系統相關。
  - `/api/web/...`：網站管理相關。
  - 其他依需求再增減。

## API設計
- 整體架構：基於Hono的路由系統，支援HTTP RESTful API與SSE（Server-Sent Events）即時推送。Middleware根據request.uri分析domain，並隔離資料庫連線。
- HTTP API：
  - 結構：端點如 `/api/pages`、`/api/media`，middleware動態處理domain隔離。
  - 方法：標準CRUD（GET/POST/PUT/DELETE），支援查詢參數與JSON body。
  - 認證：使用JWT或session，整合RBAC（角色基於權限控制存取）。
  - 範例端點：
    - GET `/api/sites`：取得站點資訊（系統級或按domain）。
    - POST `/api/pages`：建立頁面。
    - PUT `/api/media/{id}`：更新媒體。
- SSE整合：
  - 端點：`/api/events`，middleware根據domain隔離事件推送。
  - 單向推送：前端用EventSource接收事件，支援按domain命名空間隔離。
  - 事件類型：`task-progress`、`notification`。
- 錯誤處理：統一JSON格式回應（錯誤碼、訊息、traceId），避免洩露敏感資訊。
- 安全性：速率限制（後續補）、CORS設定、輸入驗證（避免注入）。
- 擴展：未來支援GraphQL作為替代，但優先HTTP+SSE。

## 人員
- 系統管理員：該專案的部署者，可以修改系統設定。
- 網站管理員：在這個系統下申請一個網站並成為該網站的管理員，可以管理該網站下的所有資料。
- 其他人員：由該網站的 角色 來限制，角色可以設定各種權限。

## 安全
- AI 查詢安全：
  - 白名單表/欄；
  - 避免任意 `WHERE`/腳本注入；
  - 查詢長度、結果筆數、時間限制；
  - 記錄 `_traceId` 以追蹤。
- 媒體白名單：拒絕 SVG，上傳限制 MIME/副檔名；影像、影音大小上限。
- Secrets：環境變數注入（不入庫）。

## CI/CD 與部署
- Docker 為發佈基礎。
- 後續可加：lint/test/build、容器掃描、分環境 config（dev/stage/prod）。

## 開發風格（中文命名）
- 允許 class/變數/參數用中文；同時保留英文欄位以利資料交換。
- 範例：
```ts
class 網站資訊 {
  網域!: string; // domain
  名稱: MultilingualString;
  描述: MultilingualString;
  佈景主題: string; // theme id
  配色: string; // palette id
  骨架: string; // layout id
  建立時間: Date = new Date();
}
```
- Log格式：統一使用 `[filename] message` 格式，方便定位錯誤來源。例如 `console.error("[surrealdb.ts] SurrealDB query failed: 500")`。

- 自行設計檔案路由器（請盡量設計成容易抽離成獨立套件）

- 確立 AI 查詢白名單與驗證層（含速率限制）。
