## 目標
- 支援多站點（以 domain 區分），以 @dreamer/dweb 為核心框架。
- 資料庫：系統資料庫用 KV，站點資料用 SurrealDB（namespace 區隔）。
- 媒體：MinIO 儲存圖片/影片/audio（不含 SVG）。
- AI：預設本地 ollama，語言處理用 jsr:@dui/smartmultilingual。
- UI：AI 參照佈景主題（配色+骨架）生成，搭配 unoCSS；
- 發佈：docker。
- 文件：Markdown 為主。
- 資料查詢：自然語言轉 AI 查詢。

## 多租戶/站點隔離
- 網站識別：`domain` 為站點 ID。
- SurrealDB namespace：
  - `system`：系統資訊、通用字典、佈景、配色、骨架。
  - `site_{domain}`：站點內容資料。
  - 若未來要簡化 namespace，可改用單一 namespace，表名加前綴 `site:{domain}::表名`。
- KV：僅放全站共用資料，鍵範例：`global:{類別}:{key}`。
- 權限：以 `domain` 控制站點資料與媒體存取範圍。

## 資料模型（無 Schema）
- SurrealDB 不建立 schema；在應用層以 class 定義欄位、驗證與預設值（支援中文命名）。
- 建議欄位：`_id`、`_v`（模型版本）、`_createdAt`、`_updatedAt`、`_createdBy`、`_updatedBy`、`_traceId`。
- AI 查詢白名單
- 範例（中文 class）：
```ts
class 單字 {
  資料:MultilingualString;
  更新時間: Date = new Date();
}
```

## 媒體庫（MinIO）
- Bucket（少量固定）：
  - `media`：原始媒體。
  - `thumb`：縮圖。
- 物件 key 結構：
  - 原檔：`{domain}/{type}/{yyyy}/{mm}/{dd}/{uuid}.{ext}`
  - 縮圖：`{domain}/thumb/{sizePreset}/{yyyy}/{mm}/{dd}/{uuid}.webp`
  - `type`：`image|video|audio|file`
- Metadata 建議：`content-type`、`original-name`、`sha256`、`width`、`height`、`duration`（影音）。
- 存取：內部以服務帳號並限制 prefix；對外用 pre-signed URL（短時效）。未來可掛 CDN。

## 媒體處理
- 需求：上傳後轉一張縮圖（建議 WebP）。
- 流程：上傳 → 觸發任務（queue/async）→ 轉換 → 存 `thumb` bucket。
- 尺寸：
  - 單尺寸：`thumb-800`（寬 800px，等比縮放）。
  - 可預留多尺寸：`thumb-sm` 400px、`thumb-lg` 1200px。
- 資料庫紀錄：原檔 URL、縮圖 URL、尺寸資訊。

## 快取與邊緣（備注）
- 現階段不實作；後續可：
  - KV 作配置/佈景快取；
  - 媒體加 ETag / Cache-Control；
  - 接 CDN（靜態與媒體）。

## 搜尋
- 全站/多站搜尋交由 AI；保留在資料中存放摘要/embedding 的欄位以供未來 RAG。

## 備份/復原（備注）
- 後續規劃：SurrealDB dump、KV export、MinIO 版本控管或跨區備援。

## 數據稽核（備注）
- 後續規劃：存取審計、操作日志、站點隔離下的查詢稽核。

## UI/佈景/元件
- 佈景：系統層（KV/SurrealDB `system` namespace）存放配色、骨架、樣板。
- 元件來源：
  - 可存 DB（方便版本/審核），或檔案庫；AI 生成時僅允許白名單元件與 unoCSS 原子類。
- 生成流程：AI 讀取佈景/元件 → 產出 UI（中文命名 class/變數可用）→ 透過 unoCSS 實體化。
- 可存取性：保持語義化、鍵盤可用、WCAG 基礎。

## 路由與授權
- Domain 綁定：`domain` 映射站點設定（存於 `system` namespace 的站點資料表）。
- 權限：站點擁有者/編輯/訪客；API/內部服務需 RBAC 與速率限制（後續補）。

## 安全
- AI 查詢安全：
  - 白名單表/欄；
  - 避免任意 `WHERE`/腳本注入；
  - 查詢長度、結果筆數、時間限制；
  - 記錄 `_traceId` 以追蹤。
- 媒體白名單：拒絕 SVG，上傳限制 MIME/副檔名；影像、影音大小上限。
- Secrets：環境變數注入（不入庫）。

## CI/CD 與部署
- Docker 為發佈基礎。
- 後續可加：lint/test/build、容器掃描、分環境 config（dev/stage/prod）。

## 開發風格（中文命名）
- 允許 class/變數/參數用中文；同時保留英文欄位以利資料交換。
- 建議：對外 API/資料庫欄位仍偏英文；程式內部/模型可中文提升可讀性。
- 範例：
```ts
class 站點設定 {
  表名 = "site";
  網域!: string; // domain
  標題?: string;
  佈景代碼?: string; // theme id
  配色?: string; // palette id
  建立時間: Date = new Date();
}
```
