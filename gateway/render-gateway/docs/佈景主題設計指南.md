# 🎨 佈景主題設計指南（AI 2026 高度解耦版專用提示詞）

> **【重要】** 本文件定義了系統的視覺與動態水庫。AI 在建立新主題、理解配色或設計方塊外觀時，必須明白方塊的屬性是如何與本系統的六大模組產生全自動聯動的。

---

## 1. 六模組聯動核心架構

全站視覺徹底解耦，**佈景主題本身只是一個指針（ID）容器**，絕不內嵌具體數值。

> 佈景主題 = 骨架 (幾何) + 配色 (色彩) + 風格 (五態) + 動畫 (動效) + 裝飾 (掛件) + 圖示集 (圖示組合)

| 模組名稱 | 職責與守備範圍 | ID 前綴範例 | 與方塊的聯動方式 |
|:---|:---|:---|:---|
| **骨架 (Skeleton)** | 掌管全局的幾何 Token（間距、圓角、字級、佈局方塊） | `骨架:骨架:` | 方塊 JSON 中的 `rounded: "md"` 會自動被映射為 CSS 變數 `--radius-md`。 |
| **配色 (Palette)** | 掌管全域 OKLCH 色彩標準（主/次/強調/背景/功能色） | `配色:配色:` | 當方塊指定 `color: "primary"` 時，UnoCSS 會自動調用 `--p` 變數發電。 |
| **風格 (Style)** | 掌管 **CSS class 組合拳**（封裝 active/inactive/hover/selected/focus 五態） | `風格:風格:` | 方塊智慧五態所產生的 `[data-active]` / `[data-hover]` / `[data-selected]` / `[data-focus]` 屬性選擇器，會全自動套用此處的 class 字串。 |
| **動畫 (Animation)** | 掌管萬能動畫映射水庫（格式為 `元件:動作`） | `動畫:動畫:` | 前端組件直接透過 `動畫.配置["下拉選單:開"]` 讀取動效類名。 |
| **裝飾 (Ornament)** | 掌管頁面掛件、浮水印與 SVG 徽章路徑 | `裝飾:裝飾:` | 容器渲染時自動在對應幾何位置塞入對應的圖示 ID。 |
| **圖示集 (Icon Set)** | 掌管風格化圖示 ID 組合（不存 SVG 實體，僅存引用） | `圖示集:圖示集:` | 方塊透過 `圖示: "圖示集:圖示集:經典外框"` 指定使用的圖示集合，元件運行時動態載入對應 SVG。 |

## 2. 配色模組：OKLCH 廣色域標準

色彩值一律使用 **OKLCH 格式**（`"L C H"` 三個數字，空格分隔），全面對齊 CSS Color Level 4 標準。

> 💡 **AI 開發須知**：當你看到方塊中使用 `bg-current`、`text-current`、`border-current` 等類名時，這些 current token 會在運行時被系統自動替換為對應主色的 OKLCH 變數。

### 配色 JSON 規範範例

```json
{
  "id": "配色:配色:霓虹電音",
  "名稱": {"zh-tw": "霓虹電音"},
  "主色": "60% 0.32 310",     // 富有科技感的電子霓虹粉 (--p)
  "次色": "40% 0.28 270",     // 深邃紫 (--s)
  "強調色": "85% 0.22 140",   // 螢光綠 (--a)
  "中性色": "95% 0.01 280",   // 文字與基色
  "背景色": "12% 0.02 280"    // 暗色背景 (--b1)
}
```

## 3. 風格模組：消滅方塊 className 混亂的功臣

風格是 **五態 CSS 類名組合拳**。它是解決 AI 瞎寫方塊外觀的最終解法。方塊晶片在偵測到 `data-active`、`data-hover`、`data-selected` 和 `data-focus` 狀態時，會透過 UnoCSS 的規則自動去匹配風格配置中的類名。

### 風格核心 Key 設計規範

| Key | 觸發條件 | 用途 |
|:---|:---|:---|
| `active` | `data-active="true"` | 方塊處於激活／供電狀態時套用的類名 |
| `inactive` | `data-active="false"` | 方塊處於斷電／停用狀態時套用的類名 |
| `hover` | `data-active="true"` + `data-hover="true"` | 方塊被滑鼠懸停時額外追加的懸停效果（token 需帶 `hover:` 前綴） |
| `selected` | `data-active="true"` + `data-selected="true"` | 方塊處於選中狀態時額外追加的效果（token 需帶 `selected:` 前綴） |
| `focus` | `data-active="true"` + `data-focus="true"` | 方塊獲得鍵盤焦點時額外追加的效果（token 需帶 `focus:` 前綴） |

> **前綴規範**：`hover`、`selected`、`focus` 為疊加態（在 active 基礎上疊加），其 token 必須使用與 UnoCSS 一致的前綴命名慣例——`hover:bg-current`、`selected:bg-accent`、`focus:ring-current`。系統會在 `generateNonCurrentCSS` 中自動剝離前綴後生成 CSS，`selected` 額外搭配 `forceImportant` 壓制底層 `cube-color-*` 供電。`active` 與 `inactive` 為基礎態，不使用前綴（直接寫原子類名）。

### 風格 JSON 規範範例（外框線風格）

```json
{
  "id": "風格:風格:未來外框線",
  "名稱": {"zh-tw": "未來外框線"},
  "配置": {
    "active": "bg-transparent text-current border-2 border-solid border-current shadow-sm",
    "inactive": "bg-transparent text-current-30 border-2 border-solid border-current-10 opacity-40 select-none",
    "hover": "hover:bg-current-10/15 hover:text-current-content hover:shadow-md transition-all duration-200",
    "selected": "selected:bg-accent selected:text-accent-content selected:ring-2 selected:ring-current",
    "focus": "focus:outline-none focus:ring-2 focus:ring-current focus:ring-offset-2"
  }
}
```

## 4. 骨架模組：幾何 Token 命名法

骨架掌管全站的幾何縮放比例。前端與方塊晶片直接透過特定的前綴消費這些 CSS 變數。

### Token 規範對照表

AI 在方塊的 `args` 變體（variants）補完或 `defaults` 中指定幾何規格時，請參考以下標準映射關係：

| 方塊引數值 | 骨架配置 Token Key | 預設 CSS 變數效果 |
|:---|:---|:---|
| `rounded: "sm"` | `radius-sm` | `0.25rem`（小圓角） |
| `rounded: "md"` | `radius-md` | `0.5rem`（標準圓角） |
| `padding: "md"` | `spacing-md` | `1rem`（舒適內距） |
| `gap: "lg"` | `spacing-lg` | `1.5rem`（子元素大間距） |

## 5. 圖示集模組：語義鍵位對照表

圖示集是第六金剛，與既有 `圖示` 模型（存 SVG 實體）不同——**圖示集用標準語義鍵位映射圖示 ID**，不內嵌 SVG。

> `圖示庫（圖示.ts）= 素材池，存 SVG 實體`
> `圖示集（圖示集.ts）= 主題播放清單，14 個標準鍵位 → 圖示 ID`

### 為什麼要硬編碼鍵位？

前端元件依賴固定的語義鍵名來查詢圖示（如導覽按鈕查 `關閉`、對話框查 `確認`）。如果每個圖示集可以自由定義鍵名，元件就無法通用。標準鍵位由系統定義，保證所有圖示集對前端都是可互換的。

> **取得標準鍵位清單**：呼叫 `GET /api/v1/icon-set/keys` 可取得當前系統定義的所有標準鍵位（含數量）。AI 在生成圖示集時，必須為每個鍵位填入圖示 ID。

### 圖示集 JSON 規範範例

```json
{
  "id": "圖示集:圖示集:經典外框",
  "名稱": {"zh-tw": "經典外框", "en": "Classic Outline"},
  "描述": {"zh-tw": "經典外框線條圖示集", "en": "Classic outline icon set"},
  "圖示映射": {
    "首頁":   "圖示:圖示:首頁",
    "選單":   "圖示:圖示:選單",
    "新增":   "圖示:圖示:新增",
    "關閉":   "圖示:圖示:減法",
    "確認":   "圖示:圖示:登入",
    "取消":   "圖示:圖示:登出",
    "搜尋":   "圖示:圖示:資訊",
    "使用者": "圖示:圖示:使用者",
    "設定":   "圖示:圖示:鑰匙",
    "主題":   "圖示:圖示:調色盤",
    "配色":   "圖示:圖示:調色盤",
    "骨架":   "圖示:圖示:web_cube",
    "圖示":   "圖示:圖示:web_cube",
    "圖示集": "圖示:圖示:資料庫"
  },
  "可刪除": false
}
```

### 查詢可用圖示集

- `GET /api/v1/icon-set/all` — 回傳所有圖示集清單（名稱 + 已填鍵位數，不含完整映射）
- `GET /api/v1/icon-set/{id}` — 取得單一圖示集的完整鍵位映射
- `GET /api/v1/icon-set/keys` — 取得系統定義的標準鍵位清單（動態擴充，AI 必須為每個鍵位填入圖示 ID）

## 6. AI 嚴格禁令

### 嚴禁內嵌數值
建立新的「佈景主題.json」時，絕對不要把顏色代碼、圓角像素直接寫在主題裡，必須使用 ID 指向對應的六大模組。

### 嚴禁破壞 Open-Ended Map 結構
當需要為新組件擴充動畫（如 `折疊面板:開`）或裝飾位置時，直接在對應模組的 `配置`（`Record<string, string>`）中追加 Key-Value 即可，不要嘗試修改或要求開發者修改底層 Model 程式碼。

### 嚴禁使用不存在的 CSS Class

> **取得可用 CSS class 清單**：呼叫 `GET /api/v1/unocss/classes` 可取得當前網站所有可用 UnoCSS 自訂 class 的完整列表。AI 在設計任何 `className` 時，只能使用 API 回傳的 class 名稱，嚴禁憑空編造。

- 所有自訂 class：`GET /api/v1/unocss`
- AI 可用 class 列表：`GET /api/v1/unocss/classes`
- 主題資訊：`GET /api/v1/unocss/info`

### 嚴禁 Hardcode 顏色
在設計任何風格或動畫類名時，一律透過代入 `text-current` 或 UnoCSS 內建的變數，把色彩主導權完全交給配色模組。

### 嚴禁隨意使用圓角
圓角大小由**骨架模組**統一管轄，而非由單一方塊自訂。AI 必須遵守以下分級原則：

- **`sm`**：緊湊元件（如小型按鈕、標籤、輸入框）
- **`md`**：一般元件（如標準按鈕、卡片、容器）
- **`lg`**：大型元件（如大型面板、區塊容器）

> ❌ 嚴禁隨意使用 `rounded-none` 或 `rounded-full`。方塊的圓角並非絕對值——avatar 不一定是圓形、卡片不一定是方角——一切交由骨架定義的 `radius-{size}` Token 驅動。

此外，若元件本身有 `size` 引數會切換整體尺寸（如 `sm` / `md` / `lg`），則 `rounded` 也必須作為 `size` 變體的一部分同步切換，確保小按鈕配小圓角、大卡片配大圓角，而非寫死固定值。
