# 📄 頁面設計指南（AI 生成專用提示詞）

> **【重要】** 本文件是全站頁面（Page）JSON 圖紙的最高指導原則。AI 在生成、修改任何頁面時，必須嚴格遵守以下 Schema、架構分工與禁令。

---

## 1. 頁面 JSON Schema

```typescript
{
  路徑: string;                              // URL 路徑，如 "/"、"/about"、"/blog/{slug}"
  標題: MultilingualString;                  // { "zh-tw": "首頁", "en": "Home" }
  描述?: MultilingualString;                 // SEO 描述
  內容: PageContent;                         // 頁面內容結構
  方塊?: string;                             // 指向根容器的方塊 ID（預設 "方塊:方塊:容器"）
  已發布?: boolean;
}

// PageContent 格式（直接對齊 Cube 渲染器）
interface PageContent {
  direction?: "col" | "row";   // 排列方向，預設 col
  gap?: "xs" | "sm" | "md" | "lg" | "xl";   // 子元素間距
  padding?: "xs" | "sm" | "md" | "lg" | "xl"; // 內距
  align?: "start" | "center" | "end" | "stretch";
  justify?: "start" | "center" | "end" | "between" | "around";
  children: PageChild[];        // 頁面子方塊陣列（必填，不可為空）
}

// PageChild = Cube 物件（用 from 指定方塊 ID）
interface PageChild {
  from: string;                 // ⚠️ 必填！方塊 ID 或原生標籤
  // ... 其他屬性為該方塊的 args（如 variant、color、children 等）
}
```

## 2. 頁面生成核心規則

### 2.1 內容格式（直接對齊 Cube）

頁面的 `內容` 欄位**直接就是一個容器 Cube 的定義**，由 `Cube.tsx` 萬用渲染器統一處理。不需要額外的包裝層。

```json
{
  "標題": { "zh-tw": "我的網站", "en": "My Site" },
  "內容": {
    "direction": "col",
    "gap": "lg",
    "children": [
      { "from": "方塊:方塊:導覽列", "color": "primary" },
      { "from": "方塊:方塊:卡片", "variant": "elevated", "children": [...] },
      { "from": "方塊:方塊:頁尾", "color": "neutral" }
    ]
  }
}
```

### 2.2 children 中的每個項目必須有 `from`

這是 AI 最常犯的錯誤：忘記在 children 子元素中加上 `from` 欄位。

```json
// ❌ 錯誤
{ "children": [{ "text": "Hello" }] }

// ✅ 正確
{ "children": [{ "from": "span", "text": { "zh-tw": "Hello" } }] }
```

### 2.3 children 不可為空

頁面至少要有一個方塊。`children: []` 會導致空白頁面。

## 3. 使用既有方塊（非原生標籤）

AI 應優先使用系統既有的方塊，而非從零創造。

> **取得完整方塊清單**：呼叫 `GET /api/v1/cube` 可取得當前網站所有可用方塊的完整清單（含 id、名稱、描述、args 等）。在設計頁面前，應先查詢有哪些方塊可用，再根據實際清單選用。不確定方塊是否存在時，優先查 API 而非憑空猜測。

常用方塊 ID：

| 方塊 ID | 用途 | 常用 args |
|---------|------|-----------|
| `方塊:方塊:容器` | 通用佈局容器 | direction, gap, padding, rounded, color |
| `方塊:方塊:卡片` | 內容卡片（有 body slot） | variant (elevated/bordered/ghost), color |
| `方塊:方塊:導覽列` | 頂部導航 | color, 品牌文字, 連結列表 |
| `方塊:方塊:頁尾` | 底部頁尾 | color |
| `方塊:方塊:按鈕` | 互動按鈕 | 文字, color, size |
| `方塊:方塊:標題文字` | 標題元件 | 文字, level (h1/h2/h3) |
| `方塊:方塊:網格` | 網格佈局 | cols, gap |

> ⚠️ **不要憑空創造不存在的方塊 ID。** 例如 `"from": "方塊:方塊:側邊欄"` — 如果系統中沒有這個方塊，渲染會失敗。不確定有什麼方塊可用時，使用 `"div"` + `className` 原生標籤。

### Card 方塊的特殊規則

Card 方塊有 `body` slot。傳給 Card 的內容必須放在 `children` 中：

```json
// ❌ 錯誤 — title/content 不是 Card 的有效屬性
{ "from": "方塊:方塊:卡片", "title": "xxx", "content": "yyy" }

// ✅ 正確 — children 自動注入 body slot
{
  "from": "方塊:方塊:卡片",
  "variant": "elevated",
  "children": [
    { "from": "h3", "text": { "zh-tw": "標題", "en": "Title" } },
    { "from": "p", "text": { "zh-tw": "內容", "en": "Content" } }
  ]
}
```

## 4. 多國語言規範

### 4.1 所有對外文字必須同時提供 zh-tw 和 en

```json
// ❌ 錯誤
"標題": "我的網站"

// ✅ 正確
"標題": { "zh-tw": "我的網站", "en": "My Site" }
```

### 4.2 text 屬性的格式

```json
// 簡單文字
"text": { "zh-tw": "歡迎", "en": "Welcome" }

// SmartContent（含 URL 或格式）
"text": {
  "zh-tw": "了解更多關於[我們的服務]({url})",
  "en": "Learn more about [our services]({url})"
}
```

## 5. 巢狀結構規範

### 5.1 容器層級

```json
{
  "from": "方塊:方塊:容器",
  "direction": "col",
  "gap": "md",
  "children": [
    { "from": "h1", "text": {...} },
    {
      // 巢狀容器：橫向排列兩張卡片
      "from": "div",
      "className": "flex flex-wrap gap-md",
      "children": [
        { "from": "方塊:方塊:卡片", "children": [...] },
        { "from": "方塊:方塊:卡片", "children": [...] }
      ]
    }
  ]
}
```

### 5.2 遞迴深度上限

巢狀結構不得超過 **10 層**。超過會觸發渲染保護。

## 6. AI 嚴格禁令速查表

| ❌ 嚴禁行為 | ✅ 正確做法 |
|-----------|-----------|
| `children` 子元素缺 `from` | 每個 child 必有 `from` |
| `children: []` 空陣列 | 至少放一個方塊 |
| Card 頂層放 `title`/`content` 屬性 | 內容放 `children` 中 |
| 憑空創造不存在的方塊 ID | 用既有方塊 ID 或原生標籤 |
| 文字只用一個語言 | `{ "zh-tw": "...", "en": "..." }` |
| `direction: "column"` | 用 `"col"`（對齊 Cube 變體值） |
| `children` 字串當文字 | 用 `{ "from": "span", "text": {...} }` |
| 在頁面 JSON 中寫 `<script>` 或 inline event | 禁止任何腳本 |
| 使用 `text-[#xxx]` 或 `bg-blue-500` | 用 `color` 參數（primary/secondary/...） |

## 8. Cube 子項 Fragment 包裹規範

### 8.1 多個原生 DOM 子項須用 Fragment 包裹（Hono JSX 限制）

在 JSX 中使用 `<Cube>` 時，若傳入**多個連續的原生 DOM 標籤**（`<div>`、`<span>`、`<a>` 等），Hono 會合併它們，導致 `Children.toArray` 無法正確拆開。**每個子項必須用 Hono Fragment `<></>` 包裹**。

```tsx
// ✅ 正確 — 每個子項獨立 Fragment
<Cube from="方塊:方塊:列表" context={c} divider stripe>
  <><div class="px-4 py-3">項目 A</div></>
  <><div class="px-4 py-3">項目 B</div></>
  <><div class="px-4 py-3">項目 C</div></>
</Cube>

// ❌ 錯誤 — Hono 合併連續原生標籤
<Cube from="方塊:方塊:列表" context={c} divider stripe>
  <div class="px-4 py-3">項目 A</div>
  <div class="px-4 py-3">項目 B</div>
  <div class="px-4 py-3">項目 C</div>
</Cube>
```

**不需要 Fragment 的例外情況**：

1. **子項包含 `<Cube>` 元件** — Hono 不會合併元件節點
2. **僅一個子項** — 單一子項無合併問題
3. **使用 `children` slot** — 透過 slot 注入的內容不受影響
4. **非 JSX 來源**（如頁面 JSON） — children 已是獨立陣列元素，無此問題

> **鐵律**：當你不確定時，一律用 `<>...</>` 包裹每個子項，不會有副作用。
