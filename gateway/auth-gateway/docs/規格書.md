# auth-gateway 規格書

> 最後更新：2026-07-22

---

## 1. 概述

auth-gateway 是 WebCube2027 的**認證授權 Gateway**，負責處理使用者登入、JWT Token 簽發與驗證。為其他 Gateway 提供統一的身份驗證服務。

**核心職責**：

- 使用者登入（目前支援本地帳號密碼認證）
- JWT Token 簽發與驗證
- 提供 Token 驗證 API 供其他 Gateway 內部調用
- 健康檢查代理（代理 data-gateway 的健康狀態）

## 3. 啟動流程

```
main.ts
  │
  ├── createGateway()                   ← @dui/framework
  │     ├── L1 init (l1.json)            ← 資料目錄: ./data/
  │     ├── Crypto key init (存入 L1)    ← registerKey() 註冊
  │     ├── Hono app 建立
  │     └── 檔案路由載入 (routes/)
  │
  ├── API 路由註冊（/api/login、/api/verify、/api/jwt-public-key）
  │
  ├── Ed25519 金鑰對管理
  │     ├── 檢查 L1 是否有 _jwt_private_key
  │     ├── 若不存在：呼叫 crypto.subtle.generateKey('Ed25519')
  │     │     ├── 匯出 PKCS#8（私鑰）→ hex → 存入 L1
  │     │     └── 匯出 SPKI（公鑰）→ hex → 存入 L1
  │     ├── 若已存在：從 L1 讀取 hex → importKey() 還原 CryptoKey
  │     └── 公鑰 hex 保留在記憶體供 /api/jwt-public-key 端點使用
  │
  └── gw.start()                        ← 啟動 HTTP Server
```

### 3.1 使用 `@dui/framework`

auth-gateway 使用 `@dui/framework` 的 `createGateway()` 統一處理以下初始化：

1. **L1 KV Store** — 在 `gateway/auth-gateway/data/l1.json` 建立設定儲存
2. **加密金鑰** — 自動產生 256-bit 金鑰，存入 L1 的 `_crypto_key`，透過 `registerKey()` 註冊
3. **Hono 應用程式** — 建立 HTTP server
4. **檔案路由** — 自動載入 `routes/` 目錄下的頁面路由

### 3.2 Ed25519 金鑰對管理

auth-gateway 在啟動時會檢查 L1 中是否有 `_jwt_private_key`：

```typescript
async function getOrCreateKeyPair() {
  const storedPrivateHex = await gw.l1.get('_jwt_private_key');
  if (storedPrivateHex) {
    // 從 L1 還原金鑰對
    const privateKey = await crypto.subtle.importKey(
      'pkcs8', hexToBytes(storedPrivateHex), { name: 'Ed25519' }, false, ['sign'],
    );
    const publicKey = await crypto.subtle.importKey(
      'spki', hexToBytes(await gw.l1.get('_jwt_public_key')!), { name: 'Ed25519' }, false, ['verify'],
    );
    return { privateKey, publicKey };
  }
  // 產生新金鑰對
  const keyPair = await crypto.subtle.generateKey(
    { name: 'Ed25519' }, true, ['sign', 'verify'],
  ) as CryptoKeyPair;
  const pkcs8 = new Uint8Array(await crypto.subtle.exportKey('pkcs8', keyPair.privateKey));
  const spki = new Uint8Array(await crypto.subtle.exportKey('spki', keyPair.publicKey));
  await gw.l1.set('_jwt_private_key', bytesToHex(pkcs8));
  await gw.l1.set('_jwt_public_key', bytesToHex(spki));
  return { privateKey: keyPair.privateKey, publicKey: keyPair.publicKey };
}
```

JWT 使用 **EdDSA (Ed25519)** 非對稱演算法簽章，Token 有效期為 **24 小時**。

**安全特性**：
- 私鑰只存在 auth-gateway 的 L1 中，永不外洩
- 其他 gateway 透過 `GET /api/jwt-public-key` 取得公鑰進行本地驗證
- 支援金鑰輪換：重新啟動時若 L1 金鑰被更新，會自動使用新金鑰

## 4. 資料目錄

```
gateway/auth-gateway/
└── data/
    └── l1.json              ← L1 設定儲存（_jwt_private_key、_jwt_public_key、加密金鑰等）
```

auth-gateway **不涉及 L2/L3 資料庫操作**。使用者驗證透過 data-gateway 的內部 API（`/inner-api/auth/verify-user`）進行。

## 5. API 路由

### 5.1 認證 API

| 方法 | 路徑 | 說明 |
|------|------|------|
| POST | `/api/login` | 使用者登入（驗證帳號密碼，簽發 JWT） |
| GET | `/api/jwt-public-key` | 取得 Ed25519 公鑰（hex 編碼 SPKI 格式），供其他 gateway 本地驗證 |
| POST | `/api/verify` | Token 驗證（要求 body 傳入 token） |
| GET | `/api/verify` | Token 驗證（從 Authorization Header 或 Cookie 讀取 token） |

### 5.2 登入流程

```
POST /api/login
  │
  ├── localProvider.login(c)           ← 驗證帳號密碼（透過 data-gateway inner API）
  │     └── fetch POST /inner-api/auth/verify-user
  │
  ├── 若驗證成功：
  │     ├── sign JWT (EdDSA/Ed25519, 24h expiry)
  │     └── Set-Cookie: jwt=... (HttpOnly, SameSite=Lax)
  │
  └── 回傳 { success, data: { token, 帳號, 角色 } }
```

**登入請求主體**：

```json
{
  "帳號": "admin",
  "密碼": "password"
}
```

**登入成功回傳**（角色為陣列格式）：

```json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJFZERTQSJ9...",
    "帳號": "admin",
    "角色": ["使用者:角色:超級管理員"]
  }
}
```

### 5.3 Token 驗證

其他 Gateway 可透過 `POST /api/verify` 或 `GET /api/verify` 驗證 JWT Token。

`POST /api/verify` 請求主體：

```json
{
  "token": "eyJhbGciOiJFZERTQSJ9..."
}
```

回傳：

```json
{
  "valid": true,
  "payload": {
    "帳號": "admin",
    "角色": ["使用者:角色:超級管理員"],
    "exp": 1234567890,
    "iat": 1234567890
  }
}
```

> 角色欄位為字串陣列，每個元素為 role ID（如 `使用者:角色:超級管理員`），支援一個使用者擁有多個角色。

### 5.4 其他 API

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/health` | 健康檢查（代理至 data-gateway 的 `/health`） |

### 5.5 檔案路由

`routes/` 目錄下的頁面路由由 `@dui/framework` 的 `createFileRouter()` 自動載入：

| 路徑 | 檔案 | 說明 |
|------|------|------|
| `/` | `routes/index.tsx` | 首頁 |
| `/login` | `routes/login/index.tsx` | 登入頁面 |

## 6. 使用者驗證流程

auth-gateway 本身**不儲存使用者資料**，使用者驗證委託給 data-gateway。使用者資料儲存在 `使用者` collection 中，以 model type `使用者` 區分使用者記錄與角色記錄：

```
auth-gateway                  data-gateway
    │                             │
    │  POST /inner-api/           │
    │  auth/verify-user           │
    │ ─────────────────────────► │
    │                             │── dataPool.queryByField('使用者', { field: '帳號', value }, '使用者')
    │                             │── bcrypt.compare(密碼, 密碼雜湊)
    │ ◄────────────────────────── │
    │  { success, data }          │
    │                             │
```

**使用者記錄結構**：

```json
{
  "id": "使用者:使用者:admin",
  "帳號": "admin",
  "密碼雜湊": "...",
  "角色": ["使用者:角色:超級管理員"],
  "updatedAt": "2026-07-22T..."
}
```

> 角色以 ID 陣列形式儲存，每個 ID 對應到 `使用者` collection 中的一筆角色記錄（model type = `角色`），角色名稱使用 `@dui/smartmultilingual` 的 MultilingualString 格式。

這種設計確保：
- 使用者資料統一由 data-gateway 管理
- auth-gateway 專注於認證邏輯與 Token 管理
- 日後可擴充多種認證提供者（Provider）

```typescript

  payload?: Record<string, unknown>;
  error?: string;
}

interface AuthProvider {
  login(c: Context): Promise<AuthResult>;
}
```

## 8. 安全設計

- **JWT Token**：EdDSA (Ed25519) 非對稱簽章，24 小時有效期
- **HttpOnly Cookie**：防止 XSS 擷取 Token
- **非對稱金鑰對**：Ed25519 金鑰對自動產生，私鑰以 PKCS#8 格式 hex 編碼儲存於 L1，永不外洩
- **公鑰公開**：其他 gateway 透過 `GET /api/jwt-public-key` 取得 SPKI 公鑰進行本地驗證
- **金鑰輪換**：重新部署時更換 L1 金鑰即可立即生效
- **不儲存密碼**：使用者驗證委託 data-gateway，auth-gateway 不接觸密碼資料
