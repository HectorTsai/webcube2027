# ai-gateway 規格書

> 最後更新：2026-07-23

---

## 1. 概述

ai-gateway 是 WebCube2027 的 **AI 代理 Gateway**，提供統一且與 OpenAI 相容的 API 介面，後端串接多種 AI provider（OpenAI、Anthropic、Gemini、Ollama）。

**核心職責**：

- 提供 OpenAI 相容的 REST API（`/v1/chat/completions`、`/v1/models`）
- 管理 AI 伺服器資源池（動態路由、容錯切換、併發限制）
- 透過 data-gateway 存取所有資料（無直接資料庫依賴）
- 依賴 auth-gateway 進行 JWT 驗證（無自有帳號系統）
- 定期排程任務（AI 請求重試、用量報表）

## 3. 啟動流程

```
main.ts
  │
  ├── createGateway()                   ← @dui/framework
  │     ├── L1 init (l1.json)            ← 資料目錄: ./data/
  │     ├── Crypto key init (存入 L1)    ← registerKey() 註冊
  │     ├── Hono app 建立
  │     └── 檔案路由載入 (routes/)
  │
  ├── CORS 設定（允許各 gateway 跨域）
  │
  ├── 啟動排程器 (services/scheduler/)
  │     └── 定期檢查 pending 排程任務
  │
  └── gw.start()                        ← 啟動 HTTP Server
```

### 3.1 使用 `@dui/framework`

ai-gateway 使用 `@dui/framework` 的 `createGateway()` 統一處理以下初始化：

1. **L1 KV Store** — 在 `gateway/ai-gateway/data/l1.json` 建立設定儲存
2. **Crypto Key** — 自動產生 AES-256 金鑰並存入 L1，透過 `registerKey()` 註冊
3. **Hono 應用程式** — 建立 Hono 實例
4. **檔案路由載入** — 掃描 `routes/` 目錄，自動載入所有 `.tsx`/`.ts` 路由

### 3.2 啟動後狀態

- ai-gateway **不管理安裝流程**（無 setup 頁面），安裝由 data-gateway 統一處理
- ai-gateway **無自有資料庫連線**，所有資料透過 HTTP 呼叫 data-gateway REST API
- ai-gateway **無自有帳號系統**，JWT 驗證委託 auth-gateway

## 5. 路由設計

### 5.1 公開端點（無需 JWT）

| 路由 | 方法 | 說明 |
|------|------|------|
| `/` | GET | AI Gateway 儀表板 |
| `/login` | GET | 登入頁面（meta refresh 導向 auth-gateway） |
| `/health` | GET | 健康檢查（含 data-gateway 連線狀態） |
| `/css/*` | GET | TailwindCSS 樣式 |

### 5.2 受保護端點（需 JWT Bearer Token）

| 路由 | 方法 | 說明 |
|------|------|------|
| `/api/health` | GET | 健康狀態 API（JSON） |
| `/api/v1/models` | GET | 列出可用模型（OpenAI 相容） |
| `/api/v1/chat/completions` | POST | 聊天完成（OpenAI 相容） |
| `/api/admin/servers` | GET | AI 伺服器列表（管理用） |

### 5.3 JWT 驗證流程

```
Request → _middleware.ts
  │
  ├── 公開路徑？→ 直接放行
  │
  └── 需驗證？
        ├── 本地快取有 public key？
        │     ├── 有 → 使用快取公鑰驗證
        │     └── 無 → 向 auth-gateway GET /api/jwt-public-key 取得
        │                 快取並繼續驗證
        │
        ├── 驗證成功？→ 放行，c.set('tenantId', payload.sub)
        │
        └── 驗證失敗？→ 401 Unauthorized
```

## 6. API 規格

### 6.1 POST /api/v1/chat/completions

OpenAI 相容聊天完成 API。

**Request Body**：
```json
{
  "model": "gpt-4o",
  "messages": [
    { "role": "system", "content": "You are a helpful assistant." },
    { "role": "user", "content": "Hello!" }
  ],
  "stream": false,
  "maxTokens": 2048,
  "temperature": 0.7
}
```

**Response**：
```json
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "model": "gpt-4o",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Hello! How can I help you today?"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 10,
    "completion_tokens": 20,
    "total_tokens": 30
  }
}
```

### 6.2 GET /api/v1/models

列出所有可用模型（OpenAI 相容）。

**Response**：
```json
{
  "object": "list",
  "data": [
    {
      "id": "gpt-4o",
      "object": "model",
      "created": 1686935002,

```

### 6.3 管理端點

`GET /api/admin/servers` — 列出所有 AI 伺服器設定（含啟用/停用狀態）。

## 7. 服務層

### 7.1 AI 資源池 (`services/aiService/pool.ts`)

`AIResourcePool extends BasePool<string, AI伺服器介面>`

| 功能 | 說明 |
|------|------|
| `loadServers()` | 從 data-gateway 載入所有 AI 伺服器設定（直接存 JSON，不做 class 轉換） |
| `selectServer(model)` | 根據模型名稱，選擇當前併發數最低的伺服器 |
| `dispatch(req)` | 自動路由 + 容錯（連續失敗 3 次自動冷卻） |
| `onHeartbeat()` | 每 30 秒檢查所有 adapter 連線狀態 |

**設計要點**：
- Pool 儲存的是 **`AI伺服器介面`**，不是 class 實例。從 data-gateway 載入的 JSON 直接作為介面值使用，避免不必要的轉換開銷
- **Runtime 狀態獨立**：`當前總併發`、`連續失敗次數`、`解禁時間戳` 等執行期狀態儲存在 pool 內部的 `runtime: Map<string, ServerRuntime>`，不與持久化資料混雜
- 所有伺服器以 `persistent=true` 加入池中，不被 eviction 掃除
- 每 30 秒 heartbeat 檢查連線
- 無 flush 需求（onFlush = no-op）

### 7.2 Provider Adapter (`services/aiService/provider/`)

每個 AI provider 透過 `getProviderAdapter(providerType)` 動態載入對應的 adapter，回傳 `AIProviderAdapter` 實例：

```typescript
// adapter.ts — Bridge 介面（英文方法名，Pool 層使用）
interface AIProviderAdapter {
  chat(server: AIServerInfo, req: AIRequest): Promise<AIResponse>;
  ping(server: AIServerInfo): Promise<void>;
}

// AIServerInfo — 精簡伺服器資訊（避免與 model 循環相依）
interface AIServerInfo {
  url: string;
  apiKey: string;
  [key: string]: unknown;
}
```

內部實作使用動態 import（`await import('./openai.ts')`）避免循環相依，並將各 provider 的中文 API（`聊天()`、`檢查可用性()`）橋接為上述英文介面。

支援的 provider：
- **openai** — OpenAI API 相容
- **anthropic** — Anthropic Claude API
- **gemini** — Google Gemini API
- **ollama** — 本地 Ollama API

### 7.3 dataGwClient (`services/dataGwClient.ts`)

統一的 data-gateway REST 客戶端，提供：
- `list<T>(collection, model?, options?)` — 列表查詢
- `get<T>(id)` — 單筆查詢
- `create<T>(collection, data)` — 新增資料
- `update(id, data)` — 更新資料
- `del(id)` — 刪除資料
- `health()` — 檢查 data-gateway 健康狀態

### 7.4 排程器 (`services/scheduler/index.ts`)

每分鐘執行的背景排程：

| 任務 | 說明 |
|------|------|
| `ai_retry` | 自動重試失敗的 AI 請求 |
| `usage_report` | 彙總各 AI 伺服器用量（TODO） |

資料讀寫全部透過 dataGwClient。

|------|------|

### 9.2 外部依賴

| 服務 | 用途 |
|------|------|
| **auth-gateway** | JWT Token 簽發、公鑰 API（`/api/jwt-public-key`） |
| **data-gateway** | 資料庫 CRUD（L2/L3）、健康檢查 |

### 9.3 通訊方式

```
ai-gateway ──HTTP──▶ auth-gateway
ai-gateway ──HTTP──▶ data-gateway
```

ai-gateway 不直接連線任何資料庫，全部透過 HTTP 委託 data-gateway。

## 10. 安全設計

- **無自有帳號系統**：所有認證委託 auth-gateway
- **JWT 驗證**：使用 Ed25519 非對稱演算法，本地快取 auth-gateway 的公鑰
- **金鑰輪換支援**：若 JWT 驗證失敗，清除快取重新取得最新公鑰再試一次
- **隔離資料存取**：所有資料操作需經 data-gateway，不直接暴露資料庫
